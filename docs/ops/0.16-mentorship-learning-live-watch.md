# 0.16 Mentorship And Learning Live Watch

Date: 12026-06-12
Scope: live verification checklist after the late-`0.16.x` mentorship, tracking-learning, visibility and world-tick diagnostics work.

This checklist is observational only. It does not approve new mechanics, a full following skill, scheduler/runtime tuning, group movement or broader content work before live signals are reviewed.

## Mentorship And Tracking Prompts

- Guided tracking-practice prompt frequency: does the prompt feel rare and earned after a real mentored following/route-memory moment?
- Tracking prompt copy: does it read like mentor guidance from the world, not a tutorial popup or command manual?
- Tracking prompt cooldown/dedupe: while following one mentor, repeated movement should not produce prompt spam.
- Visibility guardrails: tracking or observation learning should not happen in darkness unless local light makes the observed sign plausible.
- Rabbit movement observation: the cue should feel like watching the world move, not a farmable popup.
- Public progress surface: ordinary player UI should still avoid raw skill numbers and a public `/skills` sheet.

## Follow Assist

- Arrival behavior: once the followed target is already in the current location, follow assist should not queue an extra catch-up move.
- Failure clarity: if posture, darkness, stale route memory, hidden exits or manual queued actions block catch-up, the response should name the practical blocker.
- Scope: follow assist stays ordinary visible-route help. It is not group movement, hidden-route sharing, auto-navigation or the full `Слідування` skill.

## Runtime And Readiness Watch

- `/tickGet`: confirm the world tick runtime view shows last finished time, current phase, last phase, skipped-because-running count and stale warning correctly after deploy.
- `/health`: confirm readiness remains lightweight and does not depend on full world/stat diagnostics.
- Queue evidence: if responsiveness worsens, collect idle, active repeated-action and after-drain `/queueDebug` snapshots before proposing runtime tuning.

## Public Links And Learning Surface

- Public news/site/Herald deep links route correctly for `/sleep_tutorial`, `/follow_assist_on`, `/make_tincture` and `/brew_tincture`.
- The repeatable learning/help surface should keep command hints qualitative and short; do not widen it into a raw progression table.
- If live testing shows prompt noise or unclear mentor copy, prefer one tiny mentor-route polish PR before promoting the full following skill.

## Decision After Watch

Choose exactly one next lane from evidence:

1. One tiny mentor route/copy polish if live mentorship is mostly stable but one prompt path feels unclear.
2. Post-`0.16` content lane if responsiveness and learning prompts feel acceptable.
3. `0.17` runtime heartbeat work only if telemetry/world-tick evidence demands scheduler ownership work.
