## Live Smoke Script

Use a fresh test player or reset test account.

1. Start onboarding with `/start`.
2. Try a prepared name and a custom name path.
3. Enter the tutorial dream.
4. Move through the first tutorial locations using buttons and typed aliases.
5. Use `Озирнутися` (`/look`) and `Роздивитися` (`/examine`).
6. Gather visible resources, open `Речі` (`/inventory`), eat once and verify no action bypasses an active queue.
7. Sit, rest, stand and confirm the keyboard state updates without status-message spam.
8. Finish or leave the tutorial; verify help/profile no longer push unfinished-tutorial reminders after completion.
9. Walk into the first waking-world locations and inspect at least one feature with both `look <feature>` and `examine <feature>`.
10. Use `Повернення` (`/respawn`) while the character is still fresh enough; confirm the prompt, consequence and return location.
11. Trigger AFK/end-session controls and confirm tutorial nudges/proactive messages stay quiet.

## Post-Deploy Watch Points From Late `0.13.x`

- Inventory/fire actions now go through the action queue and may feel slower. This is expected, but live play should confirm the queue feedback stays clear.
- Freshening failure can feel harsh because a failed attempt still consumes the corpse. This matches the current survival rule, but later skill/tool progression may need to soften it.
- Natural `twigs` regeneration adds resources over real time. Watch after deploy for overly generous local `twigs` buildup despite the natural cap.
- Hunters and predators can spend more attempts on mice/rabbits now that attacks can miss. This is expected, but gate hunting should be watched so the loop does not become too slow.
