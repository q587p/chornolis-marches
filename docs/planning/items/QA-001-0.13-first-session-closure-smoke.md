# QA-001: 0.13 First-Session Closure Smoke

## Goal

Before starting `0.14.x`, manually verify that `0.13.x` delivers a stable first-session path.

This is the closure gate for `0.13.x`, not a new feature.

## Scope

Run a new-player smoke pass through:

- name onboarding;
- tutorial dream;
- first `Озирнутися`;
- first `Роздивитися`;
- gathering/inventory/food;
- sitting/resting;
- waking or finishing tutorial;
- first ordinary waking locations;
- `/respawn` / `Повернення` safety valve;
- AFK/end-session behavior around tutorial nudges.

## Out of Scope

- Day/night implementation.
- Ordinary sleep implementation.
- Hidden presence.
- Observation-learning mechanics.
- Weapon/equipment expansion.
- NPC hunger/cooking behavior.

## Acceptance

- A new player can complete or leave the tutorial without getting stuck.
- The first waking locations give enough context to continue.
- `look <feature>` and `examine <feature>` feel distinct.
- `/respawn` works for early/lost characters and does not feel like fast travel.
- AFK/end-session guards prevent repeated Telegram nudges.
- Findings are recorded as small follow-up tasks or fixed directly if trivial.

## Validation

Manual QA checklist is enough. Runtime tests are only needed if code changes are made.
