# RET-001 — Daily And Weekly World Tasks

## Goal

Add daily and weekly real-time tasks that give players a reason to return regularly without turning Chornolis into a checklist treadmill.

## Direction

- Use real-time daily/weekly cadence for availability, because the goal is return rhythm, not only in-world calendar simulation.
- Present tasks diegetically: notices, camp needs, local omens, seasonal work, small requests, or personal chronicle prompts.
- Prefer varied, low-pressure prompts over fixed grind goals.
- Let tasks point players back into ordinary world actions:
  - look/examine a changing place;
  - gather, freshen, cook, help the camp, follow a trace, return safely;
  - greet or notice a local character;
  - check a board, chronicle, omen, or visible environmental change.
- Keep rewards modest: memory, small supplies, reputation flavor, chronicle marks, or tiny practical help.

## Guardrails

- Not now and not in the first post-dream questlet slice.
- Do not require daily login streaks.
- Do not punish missed days.
- Do not make rewards large enough to become the main economy.
- Do not expose raw "daily quest" MMO framing in player-facing text.
- Do not add randomized tasks that point into unsafe/unreachable locations for beginners.
- Do not make tasks depend on hidden routes unless the character has learned or discovered them independently.

## First Slice Ideas

- One daily camp note: a small practical task near the starter area.
- One weekly world note: a broader atmospheric prompt tied to region, weather, moon circle, or chronicles.
- A scribe/admin debug view for currently offered tasks, without exposing hidden content to ordinary players.

## Acceptance Notes

- Daily/weekly tasks are optional and safe to ignore.
- The cadence uses real-time scheduling, but player-facing text stays in-world.
- Tasks integrate with ordinary actions rather than creating a parallel minigame.
- Tests cover reset timing, missed-day behavior, reward caps, and beginner-safe target selection.
