# 0.10.13 — Action queue diagnostics and latency cleanup

Technical patch for quick actions and creature backlog visibility.

## Added

- `/world` now shows action queue diagnostics: queued/running player actions, queued/running creature actions, overdue running actions and oldest queue delay.
- The HTTP status page and `/health` JSON now expose the same action queue diagnostics.

## Changed

- `/tickGet` now formats sub-second quick actions as `0,1 с` instead of rounding `100ms` up to `1 с`.
- The action loop starts due/queued player work before creature backlog and uses a smaller completed-creature batch, reducing the chance that animals make player actions feel stuck.
- World ticks now preload active creature action IDs once instead of counting active actions separately for every living creature.

## Fixed

- Quick player actions with stamina above 0 were already configured as `100ms`; the visible timing text now matches the actual value.
