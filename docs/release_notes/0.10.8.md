# 0.10.8 — Timing and action queue fixes

## Summary

This patch makes player actions feel responsive again and fixes due movement actions that could sit in `RUNNING`.

## Fixed

- The action queue loop now runs immediately on start/restart.
- The action queue loop now uses the current runtime poll interval after `/tickSet`.
- Due movement actions should no longer remain stuck in `RUNNING`.
- Player actions are processed ahead of creature backlog, so animal/NPC activity should not delay player movement.
- Location features no longer render with an extra list dash before the feature icon.
- Compact exit labels now render with spaces, for example `Зх Сх` instead of `ЗхСх`.

## Timing

- Quick player actions now take 0.1s.
- Action/recovery polling now runs every 0.1s, so quick actions can complete promptly.
- Rest stamina recovery is smoother: +1 every 4 ticks while resting instead of one large +42 jump every 40 ticks.
- Passive stamina recovery remains +1 every 40 ticks.
- Tired/non-quick actions are shorter: the base multiplier is now 3 ticks instead of 9.
- Tired/non-quick attacks now take 2x movement time while keeping their higher stamina cost.
- `/tickGet` now clarifies that listed action durations apply during fatigue or to creatures without quick mode.

## Technical

- Bumped package metadata to `0.10.8`.
