# 0.11.1 — Creature queue throughput

Patch release focused on reducing creature action backlog.

## Changed

- Creature action completion now processes a larger due batch with bounded concurrency.
- Routine animal movement/gathering no longer writes a world event for every completed creature action.
- Location target keyboards are paginated, with numbered duplicate creature labels for crowded places.
- Corpse target cards now include `Підібрати`; picked-up corpses go into `Речі` but continue decaying through the world lifecycle.
- Greeting/social messages now have separate actor, target and observer text, with the spoken line rendered as a quote block.
- Render deployment docs now list `CREATURE_RUNNING_ACTION_BATCH` and `CREATURE_COMPLETION_CONCURRENCY`.

## Operational Notes

After deploy, check `/world` for `overdue` and `max overdue` trends. If they still grow, the next optimization should reduce creature action generation or bulk-complete simple creature actions.
