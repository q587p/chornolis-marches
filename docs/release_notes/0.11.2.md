# 0.11.2 — Player action responsiveness, chat log and UI terminology

Patch release focused on keeping player actions quick while the living world is busy, and on making speech, inspection and labels feel more like Chornolis.

## Added

- Added `/chat` for admins with paginated recent speech from the world log.
- Added web `/chat` and `/chat.json` endpoints alongside the status pages.
- Detailed `/examine` now includes fresh movement tracks found in the current місцина.
- The main character keyboard button now shows qualitative `Життя` and `Снага` state.

## Changed

- Player action completions now run separately from the creature queue so a large animal backlog should not delay quick player actions.
- `/say` and NPC speech now render the spoken line as a Telegram quote block.
- After a player kills an animal, the bot immediately shows corpse actions such as pickup and freshening.
- Touched player-facing UI now prefers `Життя` and `Снага` instead of `HP` and technical stamina wording.
- Corpse items now display with better Ukrainian forms such as `труп зайця`, including sex-aware rabbit forms where known.

## Fixed

- Queue clear/cancel refreshes the main keyboard when the queue becomes empty.
- Old corpse item keys now have readable Ukrainian fallback names.

## Planning

- Added backlog/icebox notes for visible pickup and gathering feedback, interactive map-feature actions, localization, vegetation recovery, resource ecology and deeper predator behavior.
