# 0.11.7 — Local detail mode, keyboard UX and public web status

Release date: 12026-05-27

## Gameplay / UI

- Main movement buttons now live in the persistent reply keyboard instead of under the location message. The keyboard shows available north/west/east/south exits, `Озирнутися`, `Роздивитися`, `Меню`, an inventory button when the character carries items, and a bottom life/stamina strip.
- The life/stamina strip is qualitative by default and shows exact `Життя` / `Снага` numbers only for scribes/admins who enabled local technical details.
- Added a dedicated `Речі` inventory view. `/inventory`, `речі` and `інвентар` open the same list, and the character card now links to it instead of printing carried items inline.
- Added public `/who` and the `Хто активний` menu button for characters active in the last real hour. Scribes are listed separately; other players and NPCs are mixed alphabetically.
- Added public web `/who` and `/who.json` with the same active-character list and summary counts.
- Added `/playerAdmin <#id|name|username>` for scribes to inspect player details from anywhere; Telegram `/all` now includes player detail buttons for the current page.
- Character and player-inspection views now hide exact life, stamina, hunger and statistics unless the viewer is a scribe/admin with local technical details enabled. Ordinary player text uses visible states such as standing/sitting, rested/tired and hunger intensity.
- Detailed location inspection now hides coordinates and danger unless the viewer is a scribe/admin with local technical details enabled.
- Player hunger is capped to the current simple 13-point scale for new action updates.
- Empty money appears as `немає` instead of listing `0 ґривень, 0 шагів`.
- Inspecting another player no longer reveals their hunger, money or inventory, but now shows a gender-aware visible state, a simple unarmed/no-weapon line and what they visibly hold in their hands.
- Lit torches are limited to two active carried flames; if both hands already carry fire, trying to light a third torch gives a clear refusal.
- Player attack buttons stay generically labeled as `Атакувати`; successful unarmed attacks against small prey are currently described as trampling. Predator and character attacks answer that this combat is not implemented yet, and corpse follow-up actions use `Оглянути труп`.
- Feature rows no longer show technical light/rest bonuses such as `дає світло` or `відпочинок до ×5 снаги` unless the current player is a scribe/admin with local technical details enabled.
- Added scribe-only `/debugGet` and `/debugSet <0|1>` as a local per-character technical detail toggle, with `true`/`false` aliases. Scribes also get a character-card button under auto controls for toggling the same setting; the debug commands remain available directly and through `/adminHelp`, but are not added to the scribe-scoped Telegram command menu.
- Corpse inspection and `Лежить` output now use condition bands instead of exact decay ticks by default: `жахливо`, `скоро зіпсується`, `не дуже добре`, `середньо`, `ідеально`.
- Exact corpse ticks are still available for scribes/admins when local technical details are enabled.
- `/say` and text aliases now try to treat the first word or phrase as a visible speech target before accepting it as part of the sentence. For example, `/say Аїд Доброго вечора, ми з України` becomes addressed speech to Аїд when that target is visible.
- Fading torch warnings are now separate chat messages sent by the world tick when a carried lit torch reaches its final minutes, instead of being rendered inside every location view.
- Fading campfires now also send a local chat warning when their flame starts going out.
- Fading campfires now change their visible location and feature text to say the fire is burning low while still lighting the місцина.
- `/stat` now includes active player counters alongside the top predator list: hunting, successful gathering, greetings, speech and movement.
- Social signals now write `SOCIAL_SIGNAL` world events and appear in `/chat`.
- Player auto-mode can occasionally use a visible social signal when someone or something is nearby.
- Herbalist NPCs now have a small signature signal set: asking for quiet, nodding to characters, and pointing at animals.
- Living animals can now make a simple reaction to directed social signals: a startled animal may cancel what it was doing and flee to another location.
- The periodic `Світ ворухнувся` tick summary is a local technical report now and is sent only to scribe/admin players who enabled technical details.
- The lisovyk elder no longer wakes, appears or wanders into the Old Bridge region or locations with an active magical campfire.
- Added a protected web `/all` page that asks for `ADMIN_SET_SECRET` before showing the service list of players and creatures.
- Reworked the public website home page into a Ukrainian status and project overview page with `запущено`, version, active `/who` count, navigation links, `emblem-logo-01.png`, and a short description of the living-world vision and tone; the detailed world/status diagnostics moved to protected `/world` behind `ADMIN_SET_SECRET`.
- Added a temporary infinite torch stand at the closed settlement gate with `Взяти факел`.
- Trimmed the public Telegram command menu to the core entries: `/start`, `/me`, `/look`, `/menu`, `/news` and `/help`. Other commands remain available directly and through in-game buttons.

## Documentation
