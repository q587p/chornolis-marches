# 0.11.9 — Softer player text, clearer admin tools and fire cleanup

Release date: 12026-05-27

## Gameplay / UI

- `/start` now keeps an existing character where they are. Returning to the start should be handled through explicit respawn or admin movement, not by reopening the bot.
- Direction buttons refresh after movement, so the persistent keyboard better follows the exits available in the current місцина.
- `/look` keeps visible features compact, while `/examine` explains their gameplay meaning: light, rest, torch stands and fading or extinguished campfires.
- When a search finds nothing, the response can offer another search if the resource may still be present, or explain that the місцина is spent for now.
- If a target disappears, the reply offers `Роздивитися` immediately so the player can refresh their view.
- Item and action text hides small technical timing such as `(1 с)` unless local technical details are enabled.
- Inventory lines no longer show `×1` for a single carried item.

## Fire / Light

- A carried lit torch now burns out into `хмиз` instead of becoming a dry torch again.
- Inspecting another character now shows visible fire in their hands: one lit torch, two lit torches, or `Руки порожні` only when nothing obvious is held.
- Fading torch inventory text is calmer: a torch close to going out is shown as `запалений факел (скоро погасне)`, while the warning remains a separate chat message.
- The magical campfire description now states that old magic sustains it and it does not need хмиз.

## Admin / Debug

- `/chat` supports time, location and character groupings in Telegram and on the web chat page.
- `/all` exposes service buttons for visible NPCs as well as players.
- `/playerAdmin` now shows a fuller service card with exact status, resources, statistics, recent actions and name approval controls regardless of the viewer's local technical-detail toggle.
- Scribes can toggle another player's auto mode from the service card.
- Added `/teleport [character] <locationKey|x,y,z>` for moving the current or named character to a chosen місцина.
- `/addCreatureHelp` now uses current location examples and the unknown-location hint no longer points at stale keys.

## Text / Grammar

- Player and NPC statistics use gender-aware forms such as `вполював/вполювала/вполювали`.
- Player inspection text better respects grammatical gender for visible state, fatigue, posture and empty hands.
- Social signals are phrased as actions, such as someone nodding, instead of generic technical signal labels.
- `/stat` is labelled simply as `Статистика`; deeper ecology pressure stays planned as a separate view.

## Documentation / Planning

- Updated fire/light documentation for the torch-to-хмиз lifecycle.
- Added or refreshed planning notes for onboarding upkeep, gathered resource uses, starter learner cloak, teaching skills, Ukrainian-first MUD identity and future NPC/admin behavior controls.
