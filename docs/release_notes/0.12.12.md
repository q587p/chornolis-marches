# 0.12.12 - Prepared-name reservations - 12026-05-28

## Changed

- Moved the prepared character-name pool into a typed data module with explicit reservation state.
- Prepared names are hidden when reserved for future NPCs or events, not only when already used by a character.
- Expanded focused character-name regression coverage for reserved prepared names.

## Notes

- This keeps the current onboarding behavior small while moving the name registry plan one step closer to reserved event/NPC names and later database-backed content.
