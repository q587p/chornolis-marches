# 0.12.5 - Status and player action performance - 12026-05-28

This patch continues the performance lane with a narrow status-query pass and bounded player-action completion.

## Changed

- Narrowed heavy admin/status reads for `/all` and location listing pages to selected fields instead of broad relation payloads.
- `/all` now paginates creature rows at the database layer instead of loading every creature for each page.
- Added optional `STATUS_PERF_DEBUG=true` timing logs for the heavy status page builders, including success and failure paths.
- Status pagination callbacks now ignore Telegram's harmless "message is not modified" response.
- Web `/all` now builds the page before sending headers, preventing duplicate-header crashes if page rendering fails.
- Fixed the narrowed `/all` player query to match the current `Player` schema, avoiding a Prisma validation crash after the status optimization.
- Due player actions now complete with bounded concurrency instead of a fully serial completion loop, while actions for the same actor stay ordered.
- Added `PLAYER_COMPLETION_CONCURRENCY` as a runtime tuning knob for due player-action completion.
