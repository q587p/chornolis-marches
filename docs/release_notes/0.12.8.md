# 0.12.8 - Tutorial observation branch - 12026-05-28

This patch adds the first scripted proof of observation-based learning inside the tutorial dream.

## Added

- Added a side branch from the tutorial hub where players can inspect fox movement.
- Added a one-time tutorial observation event that lets the scene report a slight Tracking improvement without introducing the full skill system yet.
- Added `observe`, `watch`, `спостерігати` and `поспостерігати` as text aliases for closer inspection.
- Added a second tutorial rest room with a dream campfire that provides faster rest recovery and an above-normal stamina cap.

## Changed

- Updated tutorial hub copy and planning notes so observation sits beside the existing foraging, rest, time and safety lessons.
- Tutorial rest recovery now reads a data-driven regeneration multiplier from local features, and rest-time estimates account for it.
- The first tutorial use of `/rest` now gets a short Сон/Дрімота comment, with Codex guidance to keep future tutorial commands similarly diegetic.
- Passive and active rest stamina recovery now restore larger chunks per interval so recovery is much less sluggish during early testing.
- Dropped lit torches now remain burning ground items, light their location, can be picked back up and later burn out into `twigs` with a local message and world event.
- Brief location views now include visible pickable ground items under `Лежить`, not only corpses.
- Depleted-grass inspection text no longer repeats the same recovery warning in multiple paragraphs.
- `/restAdmin` now reports the actual saved stamina value and refreshes the main reply keyboard after use.
- Full `/stat`, the Telegram stats button and web `/stat` / `/stat.json` are now restricted to scribes/admins; a limited public ecology report remains planned as a separate diegetic surface.
