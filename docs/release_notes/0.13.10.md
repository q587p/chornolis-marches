# 0.13.10 - Hunter route plan and posture action guards - 12026-05-30

This patch adds the route-planning slice for the future NPC hunter loop and tightens a few player-facing action edges around sitting, pickup and tutorial sleep text.

## Added

- Added `src/services/npcHunter.ts` with a hunter route-plan helper.
- The helper resolves the gate carcass drop-off location and a configured magic campfire feature.
- It asks the existing route finder for a route from gate to campfire and back.
- Added hunter constants for the first five-torch bundle and one-torch return reserve.
- Added bulk loose-ground pickup for `get all`, `pick all`, `взяти все` and `підняти все`.
- Added compact `всі` pickup buttons beside loose resource stacks in location details.

## Changed

- `NPC-002` planning now records that route planning exists; the remaining work is the delayed state machine for taking torches, moving, hunting, collecting remains and returning to the gate.
- `npm test` now includes the NPC hunter helper regression script.
- Sitting now blocks physical actions that require standing, including movement, pickup, gathering, attacking, freshening, dropping, putting items into features, cooking and fire/torch handling.
- Blocked sitting actions now show a direct stand-up prompt.
- Tutorial sleep posture text now reads as sleep plus the action happening inside the dream instead of two conflicting states.
- Location and target-detail callbacks now avoid editing older description messages when the source message is no longer the latest tracked bot message.

## Validation

- Added focused tests for route directions, total travel cost, missing-route results and torch bundle constants.
- Added parser coverage for bulk pickup aliases and posture helper coverage for tutorial sleep text.
- TypeScript typecheck covers the new Prisma-backed route-plan helper.
