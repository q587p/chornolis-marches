# 0.13.11 - NPC hunter state-machine slice - 12026-05-30

This patch turns the gate hunter from route planning into a first living-world behavior slice. It is still deliberately smaller than the full torch-and-hunting MVP.

## Added

- Added `Лукан`, a seeded hunter NPC at the closed settlement gate.
- Added `tickNpcHunter()` for the first hunter state-machine path.
- The hunter routes through ordinary location exits, starts from the gate side, reaches the configured magic campfire path, seeks visible mice or rabbits, attacks through the existing delayed creature action queue, claims kills and returns them to the gate.
- Added hunter-claimed carcass helpers and tests for grouped corpse resource keys.

## Changed

- Hunter kills no longer behave like predator feeding. Claimed carcasses are hidden from ordinary target lists, carried back by the hunter state machine and deposited through `recordNpcCarcassDropoffContribution()`.
- Updated gate hunting loop and `NPC-002` planning docs with the current implementation boundary.

## Still Future

- Real NPC-held torch bundles and lit-torch switching.
- More deliberate hunting-ground route/radius tuning.
- An inspect/check beat between attack and the next decision.
- Recovery rules for claimed carcasses if the hunter is interrupted mid-route.

## Validation

- Extended `scripts/test/npc-hunter.cjs` for hunter profession detection, claimed-carcass markers and grouped corpse resource keys.
- Ran the full `npm test` suite before release metadata updates.
