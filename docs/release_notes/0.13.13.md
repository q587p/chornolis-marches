# 0.13.13 - NPC held torch inventory foundation - 12026-05-30

This patch starts moving NPCs toward player-like actor state by giving creatures their first real carried-resource table.

## Added

- Added `CreatureResource` for NPC and creature inventory.
- Unique creature seed/reset data can now include carried resources.
- `Орина` now starts with one held `lit_torch` and one spare `torch` as real carried resources.
- Added NPC torch-state helpers for carried torch count, lit-torch expiry and campfire lighting.
- NPC-held lit torches now count as local light alongside player-held and ground lit torches.

## Changed

- Hunter torch pickup from the ground writes carried resources instead of only editing `Creature.currentAction`.
- Hunter gate resupply writes a real carried torch bundle.
- Hunter torch-count decisions read real carried inventory first and only fall back to legacy `hunter_torches` markers for old runtime state.
- Visible NPC descriptions can mention an obvious held lit torch.
- Tutorial dream reply keyboards now hide `Help`, `Menu` and ordinary status buttons outside the rest lesson, using empty placeholder slots instead of exposing broad utility buttons too early.
- The first blocked attempt to move through the closed Dream Gate now gives a one-time Сон hint that the gate should be inspected.
- Dream Gate inspection accepts Ukrainian case forms such as `оглянути браму` and `придивитися до брами`.
- Failed feature/target inspection now says that the character does not see "this" here instead of describing everything as a target.
- Feature inspection now accepts the compact Ukrainian form `огл брама`.
- Unknown-command suggestions now show stable slash-command shortcuts in parentheses when available.
- `/glance` now returns only the location name and visible exits.
- The tutorial hub `Майбутні уроки` feature now uses a feather icon instead of the generic landmark marker.
- The tutorial dream now reveals the main-keyboard inventory button after pickup, gather success or explicit inventory commands.
- `NPC-004` and gate hunting loop docs now describe the implemented first slice and remaining carrying-state bridges.

## Still Future

- Claimed carcasses are still a lightweight `currentAction` bridge.
- NPC-held items do not yet cover full equipment, drops on death, hunger/food use or shared skills.
- Torch stand stock remains effectively unlimited for this slice.
