# 0.13.14 - Core loop omen and tutorial keyboard polish - 12026-05-30

This patch gathers the old-campfire memory omen slice, progressive tutorial dream keyboard polish, and gate hunting saturation follow-up planning into one release-sized update.

## Added

- Added a first old-campfire memory omen slice.
- Seeded old ordinary campfires can reveal one short atmospheric trace when fed with `twigs` or relit from a torch.
- Once revealed, the trace remains visible in later campfire inspection text.
- Added `LOOP-003` planning notes for tuning old-campfire memory omens without turning them into a ritual or reward system.

## Changed

- Tutorial dream reply keyboards now keep ordinary `Help`, `Menu`, `Status` and `Examine` controls hidden until the matching lesson has surfaced them.
- `Inventory` appears in the tutorial dream once the character actually carries something.
- Gate hunting saturation planning now records future persisted linger/cooldown, per-tick saturation caching for larger hunter counts, and the temporary plain `/put` default that should later be removed or redefined for generic containers.

## Still Future

- Old campfire memory omens are still a small first slice: more authored locations and traces can be added later.
- Tutorial lesson state still uses lightweight existing signals rather than a dedicated tutorial-progress model.
- Gate hunting saturation linger/cooldown, caching and generic container behavior remain planning items, not runtime changes in this patch.

## Validation

- Added `scripts/test/campfire-memory.cjs`.
- Extended posture/reply-keyboard checks for progressive tutorial dream controls.
- Regenerated planning exports.
- Ran the project build and full test suite for the combined branch.
