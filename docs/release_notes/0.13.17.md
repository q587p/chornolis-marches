# 0.13.17 - Small polish, status visibility and target interactions - 12026-05-30

This patch collects small 0.13.17 polish around tutorial controls, target interaction, public/admin status and command help.

## Added

- Added player-caused creature death counts to ecology/stat surfaces.
- Added split runtime status checks for HTTP, Telegram bot, database, world tick and action queue state.
- Added `/reset world`, `/reset stats` and `/reset full` admin reset modes, with plain `/reset` asking for a specific mode.
- Added targeted `look` support for visible players, NPCs, animals and features.
- Added `Look` to target interaction keyboards, plus prompt-based `Say` and `Whisper` actions.
- Added a future planning item for an independent status site that can remain available during failed builds, failed deploys or game-service outages.

## Changed

- Tutorial dream keyboards now keep the ordinary grid shape while hiding unavailable controls as placeholders.
- Returning to earlier tutorial cells no longer keeps the focused lesson keyboard.
- The Dream Gate focused keyboard no longer shows the speech phrase until the player inspects the gate.
- Public target `examine` output now avoids exact HP/stat counters unless the viewer has technical/scribe access.
- Hunter NPCs can answer direct speech through the reply flow and react to a fitting subset of social signals.
- Runtime status pages now distinguish server, Telegram bot and database state more clearly.
- `/help` is now grouped by topic, points first-time players to `/sleep_tutorial`, includes map guidance and documents `/restart`.
- The input normalizer treats underscores as spaces so clickable hints such as `/sleep_tutorial`, `/queue_cancel`, `/queue_clear` and `/auto_stop` reach the intended existing commands.

## Documentation

- Updated input-alias, social-signal, tutorial/beginner, NPC profession, location-feature, ecology, auto-mode, admin-command and lexicon docs for the 0.13.17 behavior.
- Added backlog/planning notes for future tutorial social lessons, independent status hosting, icon collision audits, hunter/herbalist follow-ups and lexicon maintenance.

## Validation

- Added or extended regression coverage for runtime status helpers, ecology death counters, admin reset modes, target keyboard layout, reply targets and input aliases.
- Ran `npm test`.
- Ran `npm run build`.
