# 0.13.18 - Hunter hardening, weapon planning and scribe audit - 12026-05-30

This patch tightens the NPC hunter loop around claimed prey, local field speech and the remaining future structure for actor carry state. It also folds in the first weapon-system planning pass and a quiet scribe-audit slice for dangerous reset tools.

## Added

- Added `docs/systems/weapons.md` and the first weapon planning items: `WPN-001`, `WPN-002`, `WPN-003`, `WPN-004` and `WPN-ICE-001`.
- Added `src/services/scribeAudit.ts` for reusable quiet scribe/admin action audit events.
- Confirmed `/reset world`, `/reset stats` and `/reset full` now write one `Scribe action: reset` event after the chosen reset finishes.
- Added `docs/systems/scribe_audit.md` with the dangerous scribe/admin command audit list.

## Changed

- Hunter-claimed carcasses now keep their `claimed_by_hunter:<id>` marker while corpse decay updates the remaining lifetime text.
- Returning hunters can keep seeing their claimed prey until it is deposited or finally disappears, instead of losing it when decay rewrites the corpse action text.
- Direct hunter field lines now use a shared speech helper that records a `SAY` event and increments the creature speech counter shown in ecology/status statistics.
- Completed the first scribe-audit planning slices and regenerated planning exports.

## Documentation

- Updated the gate hunting loop system notes with the 0.13.18 bridge hardening.
- Updated hunter-auto and actor-inventory planning items to keep real claimed-prey carry/ownership state as explicit future work.
- Added the weapon-system task index and scribe-audit system note.
