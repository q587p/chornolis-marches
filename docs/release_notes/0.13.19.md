# 0.13.19 - Session presence, quieter forest reactions and PR discipline - 12026-05-30

This patch adds the first session presence MVP, slows down abrupt forest recovery/reaction loops, and records the project workflow for starting patch/minor work on a separate branch, reviewing it through a PR into `main`, and keeping PR summaries and risk notes explicit.

## Added

- Persisted player session presence with `active`, `afk` and `ended` states.
- Stored reminder pause state and player-interaction timestamps for inactivity tracking.
- Added `AFK / away` and `End Session` controls with slash/text aliases.
- Added silent Auto-AFK after the configurable player inactivity timeout. Auto-AFK does not send its own notification.
- Added a one-idle-reminder-per-scene guard so tutorial idle nudges cannot repeat in the same unresolved prompt before Auto-AFK.
- Added shared proactive-message guards for delayed tutorial nudges, companion lines, action-queue notices, deploy notices, fire timer notices and location notifications.
- Added `/addResource`, typo-compatible `/addResourse`, `/addResourceHelp` and common resource restore shortcuts for scribe/admin location resource repair.
- Added focused session presence helper tests and AFK/end-session alias coverage.

## Changed

- Дід лісовик now wakes only after a configurable depletion delay if a region resource remains depleted.
- The direct gather-depletion path now records depletion for the world tick loop instead of waking Дід лісовик immediately.
- Default resource regeneration cadences are slower for ordinary resources, grass and exhausted locations.
- Documented that release/patch work should start on a separate `codex/` branch and open a PR into `main`.
- Added PR description expectations: concise summary, validation/checks, risks and rollback notes.
- Updated GitHub workflow docs to reflect the current docs-as-source-of-truth planning model and 0.13-0.15 milestones.
- Added `REL-001` and `SES-001` planning slices for release branch discipline and session presence.
- Updated Codex memory notes so future release work remembers branch, PR, risk and session-presence expectations.
