# 0.13.20 - Name review, admin menu, visibility and statistic polish - 12026-05-30

This patch makes character-name onboarding clearer without changing the underlying approval flow, gives scribes a faster admin menu, and tightens small command and statistic surfaces.

## Changed

- Prepared character names now explicitly say that scribes have already checked them.
- Custom character names now tell players they may continue immediately while the name still waits for later scribe review.
- The character card uses the same name-review wording, so pending custom names read as pending review instead of a hard error.
- Unknown-input suggestions now attach stable slash commands to more entries, including stat and item-use suggestions.

- Auto stand-up notices now include the matching `/stand` command hint.
- Manual and auto posture/recovery changes now notify other players in the same location when a character sits, stands, starts or stops resting, enters the tutorial dream with `/sleep tutorial`, or wakes with `/wake`.
- Reply-keyboard status labels now refresh with a short `–°—Ç–∞–Ω: ...` message when quick actions or recovery change the visible stamina/HP state.
- Movement notifications now remove stale inline controls: older `tracks` buttons are cleared when a new one is posted, and a character's arrival target button is cleared when that character leaves.
- Bulk item commands now include visible corpses: `get all`, `get all corpse`, typed filters such as `get all berries`, and `drop all [type]` work against matching carried or visible ground items, while plain `drop all` leaves held lit torches in hand.

- A tutorial dream pacing line now says "–≤—ñ–Ω/–≤–æ–Ω–∞/–≤–æ–Ω–∏" correctly instead of using an oblique pronoun in the monument/hesitation response.
- Incomplete onboarding players no longer enter a location for proactive world messages until their name is confirmed.
