# 0.13.20 - Name review, admin menu, visibility and statistic polish - 12026-05-30

This patch makes character-name onboarding clearer without changing the underlying approval flow, gives scribes a faster admin menu, and tightens small command and statistic surfaces.

## Changed

- Prepared character names now explicitly say that scribes have already checked them.
- Custom character names now tell players they may continue immediately while the name still waits for later scribe review.
- The character card uses the same name-review wording, so pending custom names read as pending review instead of a hard error.
- Unknown-input suggestions now attach stable slash commands to more entries, including stat and item-use suggestions.
- The fallback hint now points to `❔ Допомога` (`/help`) and `☰ Меню` (`/menu`) with clearer labels.
- At the tutorial gate, typing the written phrase directly now gets a Сон hint to add `сказати` / `/say`, and copied `Ви сказали: ...` echoes are accepted as speech aliases.
- Auto stand-up notices now include the matching `/stand` command hint.
- Manual and auto posture/recovery changes now notify other players in the same location when a character sits, stands, starts or stops resting, enters the tutorial dream with `/sleep tutorial`, or wakes with `/wake`.
- Reply-keyboard status labels now refresh with a short `Стан: ...` message when quick actions or recovery change the visible stamina/HP state.
- Movement notifications now remove stale inline controls: older `tracks` buttons are cleared when a new one is posted, and a character's arrival target button is cleared when that character leaves.
- Bulk item commands now include visible corpses: `get all`, `get all corpse`, typed filters such as `get all berries`, and `drop all [type]` work against matching carried or visible ground items, while plain `drop all` leaves held lit torches in hand.
- Corpse target buttons now show clean labels without decay timer text or the old colon/action suffix.
- The "deaths by characters" stat now includes non-animal NPC kills as well as player kills.
- Scribes now see `🛠 Адмін меню` (`/adminMenu`) on the main keyboard instead of the ordinary help button, with resource and fire submenus for common admin tools.
- Scribes can now give several torches or twigs at once with `/addTorch [player] [amount]` and `/addTwigs [player] [amount]`; the amount defaults to 1.
- Scribe `/addCreature` now keeps 50 as the batch size instead of a silent cap: requests such as `/addCreature mouse meadow_16_05 100 YOUNG` create all 100 in two batches, with a visible one-command cap for larger requests.
- Gate-hunting saturation now counts nearby locations by coordinates instead of only the gate's narrow technical region, so prey pressure on the bridge, riverbank and nearby grass can wake the hunter loop back up.
- A tutorial dream pacing line now says "він/вона/вони" correctly instead of using an oblique pronoun in the monument/hesitation response.
- Incomplete onboarding players no longer enter a location for proactive world messages until their name is confirmed.
- Plain text at the name-choice prompt is now treated as a custom character name, so players do not need to press the custom-name button first.
- Target lists now show a `🔪 Освіжувати всі` (`/freshen`) button when several suitable corpses can be queued for freshening one by one.
- Scribes can now use `/carcassQuest start` and `/carcassQuest stop` to force the gate carcass-dropoff hunting loop back on or into "enough for now" stand-down.
