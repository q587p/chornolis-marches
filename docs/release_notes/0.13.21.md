# 0.13.21 - Population recovery and northern forest pocket - 12026-05-31

This patch adds a quiet ecology safeguard so starter animal species cannot disappear permanently, and opens a small new northern forest pocket with a future animal-offering hook.

## Added

- `worldTick` now runs a population-floor restoration pass after animal lifecycle processing. If a starter animal species reaches `0` living, non-gone animals, the tick restores its starter living population at its starter locations.
- Population restoration is based on `src/data/starterAnimals.ts`, preserving starter species, location, count and age mix while excluding starter corpse groups.
- Technical world tick output now includes `populationFloorRestored` for system events, debug logs and scribe tick summaries.
- Added three reachable forest locations north of `forest_02_09`: `forest_02_10`, `forest_03_10` and `forest_04_10`.
- Added local berries, herbs, mushrooms and twigs to the new northern forest pocket.
- Added `Камінь малого сліду`, an inspectable animal-restoration charm feature and future hook for `ECO-005`.
- Added `scripts/test/population-restoration.cjs` to cover the starter population floor planner.
- Added predator prey-claim markers and feeding helper coverage so carnivores eat killed prey through a later `EAT` action.

## Changed

- `ECO-004` is now in testing with the first population-floor implementation.
- `MAP-003` is now in testing for its first playable expansion slice, with follow-up expansion still tracked separately.
- `ECO-005` now notes that the first shared charm feature exists, while the `/offer` action, item consumption, delayed recovery and cooldowns remain next.
- Refreshed `docs/world/world_map.md` after the map expansion.
- Updated ecology docs to describe the population-floor safeguard as quiet technical recovery rather than a full spirit/migration system.
- Predator kills no longer reduce predator hunger immediately. The corpse remains claimed and interceptable until the predator returns to eat it; if the corpse is taken first, the predator loses the meal.

## Risks

- Existing live databases need the updated seed/map data applied before the new locations and charm appear in play.
- Population-floor restoration is deliberately mechanical for now; it prevents permanent extinction but does not yet model migration, dens, spirits or regional recovery pressure.
- Restoring predators as starter animals may need balancing if a live world intentionally wants a predator species to stay gone for narrative reasons.
- The charm feature is only groundwork in this patch; players can inspect it, but the actual offering command loop is still future work.
- Predator feeding now depends on the claimed corpse remaining in place until the predator's later `EAT` action, so scavenging or player pickup can intentionally leave the predator hungry.
