# 0.13.22 - Queue visibility, AFK labels and command-menu polish - 12026-05-31

This patch makes queued work easier to see after bulk freshening and makes AFK state visible in character lists without adding new proactive notifications.

## Changed

- `freshen all` / `/freshen_all` replies now include the rendered player action queue, so the player can immediately see the queued freshening plan.
- Character cards now show a `📋 Черга` button when the character has a running/queued action or active rest.
- `/who`, the public `/who` web page and `/who.json` now append ` (відійшов)` to AFK player character names.
- `/all` now uses the same AFK suffix for player rows so scribes can distinguish active characters from paused sessions.
- Telegram side command menus now keep `/afk` as the second command, immediately after `/start`, so stepping away is easy to reach.
- Dropped burning torches now explicitly count as local light sources until their burn timer expires, and location rendering cleans up expired ground torches before showing the scene.
- Sexed animal age adjectives now agree with the displayed creature form, so male mouse labels use masculine age adjectives instead of the species-level feminine fallback.
- Scribe-scoped Telegram side command menus no longer list `/adminmenu` or the operational `/carcassquest` toggle; the commands remain available through direct input, admin help and the scribe keyboard/menu surfaces.
- Removed a stray UTF-8 BOM from `src/services/actionCompletions.ts`.
