# 0.13.23 - Target-list return context, scribe corpse setup and climbable trees - 12026-05-31

This patch smooths repeated work with large target/corpse lists, adds a scribe setup tool for carcass testing without exposing it in public news, and opens the first small vertical dry-meadow tree interactions.

## Added

- Added the scribe-only `/addCreatureCorpse <speciesKey> [locationKey|x,y,z] [count] [YOUNG|ADULT|OLD] [fresh|decaying|old]` command.
- `/addCreatureCorpse` defaults to the scribe's current location, `count=1`, `ADULT`, and `fresh`, and uses the same batching/cap behavior as `/addCreature`.
- Added a `🐾 Істоти` submenu to the scribe admin keyboard for creature keys, animal spawning, corpse setup and cleanup/test state helpers.
- Added two climbable tree features in the dry meadow, each with a real upper location and paired `/up` / `/down` movement.
- Added `/shake_tree` / `потрусити дерево` for upper tree branches; a successful shake drops 5-8 `twigs` at the base location and starts a real-time cooldown.

## Changed

- Target/corpse buttons created from paginated lists now preserve source page context.
- `↩️ Назад` from target or corpse action views returns through the same paginated target list callback, clamping naturally if the list shrank.
- `/addCreatureHelp`, `/adminHelp`, admin command docs and scribe audit notes now include the corpse setup flow.
- Marked `ADM-002` and `UX-002` as testing and regenerated planning exports.
- `/help` and `/commands` now mention vertical movement and the tree-shake action.

## Known Limits

- Tree-shake cooldown currently lives in JSON-backed `Feature.data`, so reset/reseed can reset it.
- `/shake_tree` is immediate after permission checks rather than a queued gather-like action; this is acceptable for the first slice but should move to a queued action if shaking should take time, stamina or create interruptible noise.
