# 0.13.25 - Feature inspection layers and slashless command parity - 12026-05-31

This patch makes feature inspection less repetitive and continues the command-parity rule for typed slash commands.

## Changed

- Direct feature inspection now separates brief and full detail:
  - `look <feature>` returns the compact feature detail line and a deeper inspection button;
  - `examine <feature>` keeps the richer interaction text, available actions and guide follow-ups.
- Extended slashless scribe/admin command parity beyond teleport for common typed commands:
  - `adminMenu`, `adminHelp`;
  - `addResource`, `addResourceHelp`, `restoreBerries`, `restoreHerbs`, `restoreMushrooms`;
  - `addTorch`, `addTwigs`, `addCampfire`;
  - `carcassQuest`;
  - `addCreature`, `addCreatureHelp`, `addCreatureCorpse`.
- Extended the same no-leading-slash handling to queue controls and maintenance/status commands such as `commands`, `start`, `queue_clear`, `queue_cancel`, `adminSet`, `tutorialReset`, `reset`, `debugGet`, `debugSet`, `restart`, `locationAll`, `world`, `restAdmin`, `playerAdmin`, `cleanupCreature`, `cleanupCreatures`, `forceOld`, `tick`, `tickGet` and `tickSet`.
- Recorded the existing 0.13.21 operational notes for live seed/map refresh, predator population-floor restoration and action-queue-dependent predator feeding.
- Updated `LOOP-001-A` planning notes with the first implemented two-layer feature-inspection behavior.
- Bumped package metadata to `0.13.25`.
