# 0.13.26 - Queued inventory, firewood upkeep and 0.14 planning - 12026-05-31

This patch closes several rough edges in the current survival/core-loop slice and documents the transition toward the 0.14 night/light line.

## Added

- Added queued action types for inventory and fire interactions: `USE_ITEM`, `DROP_ITEM`, `COOK`, `LIGHT_TORCH`, `DOUSE_TORCH`, `ADD_TWIGS` and `LIGHT_CAMPFIRE`.
- Added stamina costs and queue titles for those inventory/fire actions.
- Added natural `twigs` regeneration in eligible forest locations, controlled by `WORLD_NATURAL_TWIGS_*` environment variables.
- Added clickable website news archive entries on `/news`, including `?entry=` links and newer/older navigation.
- Added scribe-visible session presence reason/timestamp text for player detail pages.
- Added self-target text handling for look/examine flows such as `me`, `я`, `мене` and `мій персонаж`.
- Added 0.13 closure / 0.14 transition planning, plus small planning items for first-session QA, world-time admin safety, darkness copy, light-source matrix testing and first-night guidance.

## Changed

- Inventory buttons, text aliases and nearby fire callbacks now submit actions through the shared player action queue instead of mutating inventory/fire state immediately.
- Freshening now has species-based success chances: mouse 80%, rabbit 60%, fox/wolf 40%, and a default 50% for other species.
- Failed freshening attempts now mark the corpse as processed and produce no meat, with success/failure recorded in food-learning event descriptions.
- Lit ground torches now explicitly state that they provide light while burning.
- Brief illuminated location views can include pickup buttons for visible ground resources.
- Ukrainian verb agreement helpers now live in `src/services/grammar.ts`, with docs clarifying the boundary between the stable world lexicon and grammar machinery.
- Roadmap and near-term planning now keep `0.13.x` focused on Core Loop & Onboarding Stability and make `0.14.0` a small world-time foundation before full darkness/light/firewood/sleep work.

## Fixed

- Eating, dropping, cooking, torch and campfire actions no longer bypass an existing player action queue.
- Fire feature callbacks now share the same queued behavior as inventory buttons and text commands.
- Self-inspection no longer falls through to nearby target ambiguity when the player asks to look at themselves.

## Validation

- Ran `npm run planning:export`.
- Ran `npm run build`.
- Ran `npm test`.

## Risks / Rollback

- This includes Prisma enum additions for new `WorldActionType` values and player session presence metadata; deploy needs the included migrations before the app uses those values.
- Queueing inventory/fire actions changes timing: very small actions that used to be immediate may now wait behind the current action.
- Natural `twigs` regeneration is intentionally conservative but may need tuning per region after live observation.
- Freshening failures are now possible and may feel harsher until player skill/progression gives better affordances.
- Rollback: revert this patch and the two included migrations together if runtime queue or schema behavior needs to be backed out.
