# 0.13.27 - Core loop closure audit and attack misses - 12026-05-31

This patch closes the `0.13.x` Core Loop & Onboarding Stability lane and adds a small shared attack-miss rule for early prey.

## Added

- Added `docs/planning/audits/0.13-first-session-closure-smoke-12026-05-31.md` as the first-session closure audit report.
- Added a live-smoke script for a fresh player path through onboarding, tutorial dream, first waking locations, `/respawn`, AFK/end-session guards and scribe support checks.
- Added shared attack-hit rules for player, NPC and creature attacks.

## Changed

- Marked `QA-001` as testing with the audit report linked from the planning item.
- Updated near-term planning so `0.13.x` waits for one final live Telegram smoke after deployment, then hands off to `0.14.0` world-time foundation if no hard blocker appears.
- Attacks against mice now miss 20% of the time, and attacks against rabbits miss 40% of the time. A miss still spends stamina, marks the location as recently dangerous and leaves the target alive.
- Border marker ecology copy now reads as two compact lines for recent births/deaths and causes, instead of repeating "зарубки" in every value.

## Validation

- Ran `npm run planning:export`.
- Ran `node scripts/test/attack-rules.cjs`.
- Ran `npm run build`.
- Ran `npm test`.

## Risks / Rollback

- Attack runtime risk is moderate-low: prey survival is slightly higher, hunters/predators may need more attempts and location danger can linger from missed attacks.
- Process risk remains: if the live Telegram smoke finds a hard blocker, keep the blocker in `0.13.x` instead of starting `0.14.0`.
- Rollback: revert this patch if the miss rates or closure decision need to change.
