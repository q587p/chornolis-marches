# 0.13.6 - Speech commands and sitting rest posture - 12026-05-29

Patch release focused on finishing the near-term text command pack for speech and separating sitting posture from active rest.

## Added

- Added `whisper [player] [message]` and Ukrainian `—ą–Ķ–Ņ–Ĺ—É—ā–ł [–Ņ–Ķ—Ä—Ā–ĺ–Ĺ–į–∂] [—ā–Ķ–ļ—Ā—ā]` aliases.
- Whispered speech is private to the target player; nearby bystanders only see that a whisper happened, without the message text.
- Added `reply <message>` / `–≤—Ė–ī–Ņ–ĺ–≤—Ė—Ā—ā–ł <—ā–Ķ–ļ—Ā—ā>` for answering the latest local speech event that addressed the character by name/forms.
- Added `shout <message>` / `–≥—É–ļ–Ĺ—É—ā–ł <—ā–Ķ–ļ—Ā—ā>` / `–ļ—Ä–ł–ļ–Ĺ—É—ā–ł <—ā–Ķ–ļ—Ā—ā>` for region-level speech that spends extra stamina.
- Added a separate player posture model with `STANDING` / `SITTING`, plus `/sit`, `/stand`, `—Ā—Ė—Ā—ā–ł`, `–Ņ—Ä–ł—Ā—Ė—Ā—ā–ł`, `–≤—Ā—ā–į—ā–ł` and related aliases.

## Changed

- `/commands`, input-alias docs and `CMD-001` now list `whisper`, `reply` and `shout` as current commands.
- `docs/planning/next.md` no longer keeps `CMD-001` in the active 0.13 lane now that the compact text-command pack has shipped.
- Inventory fire actions such as lighting/dousing torches and adding twigs now send the action result as its own message, then refresh the inventory view separately.
- `/rest` now starts active recovery while sitting. If the character was standing, rest first changes posture to sitting.
- Rest completion and rest interruption clear active recovery but keep the character sitting, and standing up during rest interrupts it.
- Character and location descriptions now distinguish standing, sitting and sitting while resting.
- Main reply keyboard and character-card buttons now show `–°—Ė—Ā—ā–ł`, `–í—Ā—ā–į—ā–ł` and `–í—Ė–ī–Ņ–ĺ—á–ł—ā–ł` according to posture/rest state.

## Validation

- Parser tests cover English and Ukrainian forms for `whisper`, `reply` and `shout`.
- Parser tests cover posture aliases, and focused helper tests cover posture text and posture/rest button states.
