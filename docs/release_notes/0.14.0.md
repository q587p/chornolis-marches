# 0.14.0 - World-time foundation - 12026-05-31

This patch starts the `0.14.x` Night, Light and Firewood line with a small world-time foundation.

## Added

- Added `src/services/worldTime.ts` as the shared source for current world time and daypart.
- Added dawn/day/dusk/night helper output with deterministic test coverage.
- Added `docs/systems/world_time.md` and `docs/planning/0.14-observation-readiness.md`.

## Changed

- `/time` now reads the shared world-time helper instead of a static daytime placeholder.
- Updated planning items and near-term docs so visibility/light work follows world time, while observation-learning stays deferred to `0.15`.

## Validation

- Ran `npm run planning:export`.
- Ran `node scripts/test/world-time.cjs`.
- Ran `npm test`.
- Ran `npm run build`.

## Risks / Rollback

- Runtime risk is low: no Prisma migration, seed refresh or deploy data change is required.
- Design risk is that real-clock-derived time may later need stored or accelerated in-world time; callers should keep using `worldTime.ts` so that swap stays localized.
- Rollback: revert this patch to return `/time` to static calendar text and remove the helper/docs.
