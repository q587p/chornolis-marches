# 0.14.0 - Internal world-clock planning and prepared names - 12026-05-31

This release opens the `0.14.x` Night, Light and Firewood line as a planning and foundation release.

## Added

- Added the internal Chornolis world-clock plan:
  - `1` in-game minute = `2_000 ms`;
  - `1` in-game hour = `120_000 ms`;
  - `1` in-game day = `48` real minutes;
  - `1` lunar circle = `28` in-game days;
  - `1` in-game year = `13` lunar circles / `364` in-game days.
- Added `docs/systems/world_time.md` with the canonical starter timestamp: year `587`, `Коло Зеленого Шуму`, day `17`, `17:00` / `передвечір'я`.
- Added planning items for weather MVP state, light snapshot helper and time/weather debug controls.
- Added observation-readiness planning that keeps observation-learning in `0.15` while `0.14` builds visibility/light foundations first.
- Added proposed small core-loop planning items for a riverbank fisher moment, visible herbalist observation and a strange-track omen.
- Expanded the prepared character-name pool and its validation coverage.

## Changed

- Updated the roadmap and near-term planning so `0.14.x` is explicitly based on stored/derived internal world-clock state, not real-world time of day.
- Clarified that minimal moon phase and moon illumination belong to `0.14` because they affect night light, while deep calendar systems stay in Icebox.
- Documented reset expectations for the future world clock: world/full reset returns time and weather to the starter state; stats reset does not.
- Kept runtime day/night and weather implementation deferred to later `0.14.x` slices.

## Validation

- Ran `npm run planning:export`.
- Ran `npm test`.
- Ran `npm run build`.

## Risks / Rollback

- Runtime risk is low: the world-clock implementation is documentation/planning only in this release.
- Product risk is mostly sequencing: future code must replace the current static `/time` path with an internal stored clock, not a real-time daypart helper.
- Rollback: revert this release if the `0.14.x` milestone order or prepared-name expansion needs to change.
