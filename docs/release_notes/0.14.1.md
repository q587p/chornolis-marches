# 0.14.1 - Stored internal world clock - 12026-05-31

This patch turns the `0.14.x` world-time plan into the first runtime foundation: Chornolis now has a stored internal clock that advances through the world heartbeat.

## Added

- Added persistent `WorldState` storage for the internal world clock:
  - current absolute world minute;
  - last advancement timestamp;
  - starter weather fields reserved for the next weather MVP slice.
- Added shared world-time helpers for daypart, clock labels, lunar circle/day, moon phase and moon illumination.
- Added a focused world-time test covering the canonical starter timestamp and advancement math.

## Changed

- `/time` now reads stored/derived internal world-clock state instead of static placeholder text.
- `worldTick()` advances the stored clock by elapsed real milliseconds as a rate.
- Seed, `/reset world` and `/reset full` return the internal clock to the canonical starter timestamp.
- Attack-miss feedback now includes a quick look button so players can check whether the target stayed nearby.
- Telegram side command menus now put `/help` right after `/afk`, with `/respawn` directly below help.
- Planning docs now mark `WORLD-001-B/C/D` as the tested `0.14.1` slice and keep weather, visibility, light and ordinary sleep as later `0.14.x` work.

## Deferred

- No darkness or visibility reduction yet.
- No weather simulation yet.
- No light snapshot helper yet.
- No observation-learning, `/observe` or skill sheet work.

## Validation

- Ran `npx prisma generate`.
- Ran `node scripts/test/world-time.cjs`.
- Ran `npm run planning:export`.
- Ran `npm test`.
- Ran `npm run build`.

## Risks / Rollback

- Requires deploying the Prisma migration before running the updated bot, because `/time`, world tick, seed and reset now use the `WorldState` table.
- Existing production data needs the migration plus either seed/reset or first access to create the singleton world-state row.
- The clock advances on world tick and `/time`; if the process sleeps for a long time, the next advancement may jump the internal clock forward by the elapsed rate.
- Rollback: revert this patch and its migration if the stored-clock behavior needs to be removed before visibility/light work continues.
