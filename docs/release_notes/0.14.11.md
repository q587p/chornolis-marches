# 0.14.11 - Shared beginner cache

Date: 12026-06-01

## Summary

This release adds a small shared supply cache at the starter-camp watchtower. It gives first-session players a clearer place to take one practical supply item or leave one carried item for the next traveler, while keeping the loop narrow and diegetic.

## Player-facing changes

- Added `Спільна скриня прибулих` at `start_border_watchtower`.
- The cache can hold a small amount of torches, berries, herbs, mushrooms and twigs.
- Inspecting the cache shows qualitative stock and focused buttons for taking or contributing available supplies.
- Added text aliases around `cache`, `/cache`, `/take_cache` and `/contribute_cache`.
- Added hidden unobserved restock so the cache can recover a little after being left alone.

## Technical notes

- Cache state is stored in `LocationFeature.data` for the MVP, avoiding a schema migration.
- Restock is deliberately modest and only runs when the cache is inspected or used after being unobserved long enough.
- The watchtower torch stand remains the hunter resupply source; this cache does not rewrite hunter routes.

## Risks

- Cache stock lives in feature JSON data for now, so reset/reseed flows can reset it.
- The MVP allows one item per action and qualitative stock only; richer contribution etiquette, anti-hoarding limits and ownership rules remain future work.
