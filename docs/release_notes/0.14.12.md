# 0.14.12 - Lying posture and ordinary sleep

Date: 12026-06-01

## Summary

This release adds the first ordinary sleep MVP. Player posture now includes lying as a separate state, and plain `/sleep` starts ordinary sleep instead of routing to the tutorial dream. Tutorial sleep remains explicit through `/sleep tutorial` and `/sleep_tutorial`.

## Player-facing changes

- Added `–õ—è–≥—Ç–∏` (`/lie`) as posture-only behavior.
- Lying blocks physical actions until the player stands, while still allowing look, examine, speech and status checks.
- Plain `–°–ø–∞—Ç–∏` (`/sleep`) starts ordinary sleep in the current waking location.
- `–ü—Ä–æ–∫–∏–Ω—É—Ç–∏—Å—è` (`/wake`) wakes ordinary sleep first, leaving the player lying with sit/stand choices.
- Ordinary sleep disables auto-mode and recovers stamina faster than active rest without chatty recovery messages.

## Technical notes

- Added `PlayerPosture.LYING`.
- Added `PlayerSleepState` with `AWAKE` and `ORDINARY_SLEEP`.
- Ordinary sleep is intentionally not a posture and does not reuse `isResting`.
- Sleep cancels incompatible queued/running physical actions but does not implement world-time auto-wake yet.

## Risks

- This is the first persisted sleep state; older admin/reset paths were updated for obvious wake/reset cases, but unusual debug flows should be watched after deploy.
- Sleep quality, danger interruptions, shelter/fire comfort and world-time auto-wake are still future work.
