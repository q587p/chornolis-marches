# 0.14.13 - Sleep world-time and campfire comfort

Release date: 12026-06-01

## Summary

This patch adds the first SLEEP-003 slice. Ordinary sleep now records the internal world-clock minute when it begins, can wake automatically after enough in-world sleep time, and uses a shared recovery profile that lets active campfires improve sleep recovery.

## Player-Facing Changes

- Plain `/sleep` still starts ordinary sleep in the waking location, but sleep now tracks internal world time.
- A character can wake automatically after roughly eight game hours once HP and stamina have recovered to the relevant cap.
- Sleep has a first upper bound around ten game hours so ordinary sleep does not continue forever if recovery is incomplete.
- Active campfires improve ordinary sleep recovery and can slightly raise the temporary stamina cap.
- Manual `/wake` still works and leaves the character lying down after waking.
- The shared beginner cache at the watchtower can now show, accept and give raw meat and cooked meat as small starter-camp supplies.
- The Chancery's historical news archive backfill now keeps stable old-news order and spaces delayed archive posts instead of publishing several at once after a sleeping Render service wakes.
- Public Chancery news/archive posts can now link safe game commands such as `/look`, `/examine`, `/news`, `/auto`, `/autoStop`, `/me` and `/help` back to the main game bot.

## Technical Notes

- Added `Player.ordinarySleepStartedAtMinute` with a committed Prisma migration.
- Added `HeraldPublication.archiveOrder` with a committed Prisma migration for stable archive publication ordering.
- Added `GAME_BOT_USERNAME` config and HTML deep-link rendering for whitelisted Herald news commands.
- Added shared pure helpers for ordinary sleep duration, auto-wake decisions and sleep recovery profile signals.
- Cleared ordinary sleep timing whenever rest, wake, admin movement/restoration or beginner return forces a player awake.
- Added Herald archive backfill controls for 13-minute default spacing, one-publication-per-tick catch-up and overdue pending archive rescheduling.
- Added `scripts/test/sleep.cjs` and included it in `npm test`.
- Extended beginner-cache seed and parser coverage for raw/cooked meat.

## Risks

- Auto-wake uses the persisted world clock, so if world-clock advancement is manually frozen or reset by a scribe action, sleep duration follows that state.
- Shelter and danger interruptions are still future work; the first comfort slice only uses active campfires and existing local rest-cap signals.
- Herald archive publication ordering now depends on a new migration; deploys must run committed Prisma migrations before the updated Herald publisher handles old backfill rows.
- Herald command links are intentionally whitelist-only; new public game commands must be added deliberately before they become clickable from Chancery posts.
