# 0.14.14 - Minimal weapon equipment foundation

Date: 12026-06-01

## Summary

This release adds the first small weapon/equipped-tool layer without expanding combat scope. Players can receive and carry a starter `knife`, inspect weapon items, take one in hand, put it away, and see held weapons in character and target descriptions. Existing attack and freshening actions now read the equipped weapon where relevant, while target eligibility, damage rules and meat yield stay unchanged.

It also adds a small Herald operations guardrail: the publication queue can be paused/resumed without stopping the service, and pending news/archive rows can be canceled without deleting published history.

## Changes

- Added `Player.equippedWeaponKey` and `Creature.equippedWeaponKey` with a Prisma migration.
- Added the MVP weapon catalog and seed resource types: `knife`, `hunting_spear`, `sickle`, `hand_axe`, `short_sword`.
- Added starter `knife` grant after onboarding completion.
- Added inventory item controls for equip/unequip and matching typed aliases.
- Added weapon-aware player/creature target descriptions and location target lines.
- Added weapon-aware player attack completion and observer copy.
- Added sharp-equipped-weapon validation for corpse freshening before stamina spend.
- Added durable Herald publication queue pause/resume state.
- Added `/pause_publications`, `/resume_publications`, `/cancel_pending_publications` and `/backfill_news_cancel` for safe Herald outbox control.
- Added focused weapon helper tests and alias regression cases.

## Risks

- Equipped weapons are still keyed resources, not item instances; multiple copies of the same weapon do not track which exact item is held.
- NPC seed loadouts are not changed in this slice, so creature weapon text only appears when a creature already has `equippedWeaponKey`.
- Combat remains intentionally shallow: no durability, armor, counterattacks, PvP, wounds or weapon balance math yet.
- Canceling Herald publications marks unpublished news/archive rows as `CANCELED`; it does not delete Telegram messages or affect unrelated future publication types.
