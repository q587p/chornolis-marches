# 0.14.18 - Creature simulation budget

Date: 12026-06-01

## Summary

This release adds the first runtime scaling slice for large creature populations. World ticks now process ordinary background creatures through a configurable budget while protecting player-visible locations, non-animal NPCs and predators first.

## Player-Facing Notes

- Large waves of mice, rabbits and other background animals should put less pressure on the action queue.
- Local danger and visible nearby behavior remain prioritized: creatures around players, hunters, spirits/NPCs and predators are not treated as disposable background first.
- This is a stability and tuning step, not a full ecology rewrite.

## Technical Notes

- `WORLD_CREATURE_TICK_BUDGET` controls how many living non-busy creatures receive individual behavior per world tick. The default is `180`; `0` disables the budget.
- `worldTick()` now loads lightweight creature candidates, selects protected and budgeted ids, and fetches heavier location/resource includes only for selected creatures.
- World tick events now include `creatureBudget`, `creatureCandidates`, `creatureProcessed`, `creatureDeferred` and `creatureProtected`.
- Scribe/admin `/stat` and protected web `/stat` include the new creature tick counters for tuning.

## Validation

- Added `node scripts/test/creature-tick-budget.cjs`.
- Added the focused test to `npm test`.

## Risks

- Deferred ordinary background animals may move/eat less often when population is very high; ecology aggregate loops still run, but richer aggregate movement/eating remains a follow-up.
- The default budget is conservative and may need tuning after production observation.
- This does not yet split `worldTick.ts` into smaller load/simulate/persist phases.
