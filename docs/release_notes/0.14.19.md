# 0.14.19 - Stabilization and safety rails

Date: 12026-06-01

## Summary

This release is a stabilization pass after the 0.14 ecology, light, sleep and creature-budget slices. It adds guardrails around starter torch sources, tightens focused regression coverage and records near-term planning risks discovered during live review.

## Player-Facing Notes

- Starter torch sources now stop handing out more torches once a character is already carrying enough, and repeated rapid taking is rate-limited so supplies are not casually emptied into a pile.
- The beginner torch flow remains available as a safety rail for first-night darkness, but it now behaves more like a shared camp supply than an infinite personal generator.

## Technical Notes

- Added torch-source decision helpers with carry-limit and recent-take-window checks.
- Added notification replacement-lock tests so repeated Telegram update paths are less likely to throw noisy "message is not modified" errors.
- Expanded world seed/posture validation around starter safety rooms and map authoring.
- Updated planning exports after adding and refining planning items for sleep command gating, Herald reliability, deferred messages, profession line banks, elixirs, craftable chests, pit traps, follow intent and ecology/performance watchpoints.

## Validation

- Ran `npm test`.
- Ran `npm run build`.
- Ran `npm run planning:export`.

## Risks

- The torch-source cap is intentionally simple: it counts broad carried torch types and recent take events, not true item ownership. Later item-instance work can make this more precise.
- Recent-take limiting uses event-derived history, so unusual reset/backfill flows may not perfectly represent old behavior.
- Most changes are guardrails and planning updates; larger follow-ups such as ordinary sleep command gating, aggregate creature movement/eating and Herald outbox locking are recorded but not implemented in this release.
