# 0.14.20 - Daypart notification settings

Date: 12026-06-01

## Summary

This release adds a narrow player notification setting for routine daypart changes. Players can disable proactive dawn/day/dusk/night notices without muting urgent gameplay, direct command replies, speech, danger or queue messages.

## Player-Facing Notes

- `⚙️ Налаштування` is now available from `☰ Меню`.
- `/settings`, `/notifications` and `/daynotices` open the settings surface.
- `/daynotices off` disables routine daypart notices; `/daynotices on` enables them again.
- After the second ordinary wake-up, the game can show a one-time hint explaining that these notices can be disabled, with a warning to pay closer attention to `/time`, `/weather`, light, shadows and location descriptions.

## Technical Notes

- Added `Player.daypartNoticesEnabled`, `Player.daypartNoticeHintShown` and `Player.ordinaryWakeCount`.
- Added `src/services/playerNotificationSettings.ts` for settings rendering, toggles, daypart eligibility and one-time hint claiming.
- Added a daypart-specific proactive notification helper so the setting does not become a global mute.
- Daypart `WorldEvent` creation remains unchanged.
- Manual and automatic ordinary wake paths both use the same hint helper; tutorial wake-up does not.

## Validation

- Ran `npm run planning:export`.
- Ran `npm run build`.
- Ran `npm test`.

## Risks

- This is the first player-facing settings surface, so later preferences should grow deliberately instead of becoming a loose catch-all.
- The one-time hint is based on ordinary wake count, not on whether the player has already noticed or been annoyed by daypart messages.
- If 0.14.19 merges first, this branch may need a small version/release-doc rebase because it started from `main` at 0.14.18.
