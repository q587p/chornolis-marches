# 0.14.22 - Willow Floodplain landmarks and resources

Date: 12026-06-01

## Summary

This release fills in the first Willow Floodplain contour with inspectable landmarks, direct feature aliases and modest biome resources. It remains a narrow map/content slice: no fishing mechanics, no observation progression, no notification changes and no settlement gate changes.

## Player-Facing Notes

- Willow Floodplain locations now have more authored landmarks to inspect, including reeds, mud tracks, roots, frog water, an old net and a dark backwater.
- `Look` still gives the compact location overview, while `examine` can reveal more detail through feature summaries or direct feature inspection.
- Swamp locations now have small grass, mushroom and herb resource defaults, with no default berries.
- The dry islet has a tiny berries/herbs exception; the blind channel and dark backwater do not receive gatherable vegetation.
- Ordinary animals should no longer newly climb human-authored `UP` exits such as watchtower ladders or climbable trees.
- Active campfire feature inspections now show a direct cook-meat action when the player carries raw meat.
- Inventory and campfire controls can now queue all carried raw meat for cooking, one piece at a time.
- Edible inventory resources now offer eat-all controls and aliases that queue only the useful amount, then trim same-resource queued uses once HP, stamina or hunger is saturated.
- The admin `/all` Telegram view now separates player pages from creature pages, avoiding overlong callback edits as player counts grow.
- Inspecting someone who visibly holds a weapon or tool no longer also says their hands are empty.
- Regional shout delivery now uses the current region relation explicitly, so active nearby-region recipients are included instead of only the actor's immediate location.
- Attack observer text now uses shared visibility rules: bystanders can see the attacker's name in daylight or clear local light, while darkness can still obscure it.
- Nearby target lists and buttons no longer repeat the same visible action label when a creature has matching stored and queued actions.

## Technical Notes

- Added six `LANDMARK` features in `prisma/data/world/features.json`.
- Added feature aliases and `examine_summary` data for typed feature targeting and location-detail output.
- Added `SWAMP` resource defaults plus Willow-specific overrides in `prisma/data/world/resourceRules.json`.
- Added `MAP-WILLOW-003` and `QA-WILLOW-001` planning items and refreshed planning exports.
- Added a shared creature-exit boundary helper for animal movement and panic/flee paths.
- Added a raw-meat inventory check for active campfire feature actions.
- Added `cook all` / `/cook_all` alias routing and queued cook-all submission.
- Added `eat all` alias routing and useful-count queue trimming for edible `USE_ITEM` actions.
- Added `ECO-006` and `NPC-008` planning items for animal movement boundaries and future starter supply guards.
- Added `AUTH-001` and `ECO-007` planning notes for account recovery / client-neutral login and future animal taming.
- Added documentation follow-up notes for daypart notification settings and future weapon/torch hand-slot behavior.
- Updated the ASCII map renderer and generated map so authored climbable/shakeable tree locations render as `T`.
- Added action-label deduplication for visible target formatting.
- Kept notification settings, fishing, observation, economy, combat, theft and settlement access out of scope.

## Risks

- Feature aliases are content-driven and rely on the existing feature-target resolver; unusual phrasing may still need future alias additions after live play.
- Swamp resource amounts are intentionally conservative, but live ecology pressure may still require tuning after creatures and players spend more time in the floodplain.
- The landmarks include future hooks only as inert data/context; they do not implement fishing, track-reading, hidden routes or visibility progression yet.
- Animal-friendly vertical passages are only future-compatible in helper code; the current database schema does not persist per-exit animal-friendly metadata yet.
- Cook-all queues based on currently carried raw meat; if another action consumes that meat first, later queued cook actions will fail cleanly through the existing inventory check.
- Eat-all queues based on current inventory and vitals; if another action changes HP, stamina or hunger before completion, at most the currently running use may fail cleanly while later same-resource queued uses are cancelled.
- `/all` now paginates players and creatures as separate sections, so scribes may need one extra page turn to reach the first creature rows.
- Shout delivery still respects existing proactive/session guards; AFK, ended-session or not-yet-onboarded players remain quiet until a later deferred-message design changes that.
- Movement and attack observer labels now share visibility rules, so daylight can reveal names in places where the older active-light-only helper would have shown an unknown actor.
- The `T` ASCII-map marker depends on current feature flags (`climb_tree` / `shake_tree`); future decorative trees without those flags still render as their ordinary terrain symbol.
- Action-label deduplication is exact after whitespace and case normalization; two different phrases with the same meaning may still both show until normalized upstream.
