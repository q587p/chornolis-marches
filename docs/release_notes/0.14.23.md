# 0.14.23 - Reconciliation and safety slice

Date: 12026-06-01

## Summary

This stabilization patch reconciles several 0.14 planning slices after 0.14.22 and tightens two safety edges: ordinary sleep command gating and restart confirmation parity. It also keeps the previous clickable `shout` command polish.

## Player-Facing Notes

- Clicking `/shout` without text now reaches an explicit command handler and shows the expected format instead of falling through.
- `shout <message>` and the existing Ukrainian text aliases still use the current region-wide speech action and stamina cost.
- While a character is in ordinary sleep, active commands and active inline buttons now answer with wake-first copy instead of running through the sleeping body.
- Passive/safety commands such as `/wake`, `/time`, `/weather`, `/help`, `/commands`, `/chronicles`, `/news`, settings and session-presence controls remain available while asleep.
- `/restart` and slashless `restart` now ask for confirmation before deleting character state.
- Hunger should no longer jump after every ordinary stamina-spending action. Until the full world-time hunger model lands, it rises from strenuous actions or overexertion instead.

## Technical Notes

- Added a direct `bot.command("shout", ...)` route that reuses the existing shout submission path.
- Kept the slashless and Ukrainian alias parser behavior unchanged.
- Added a central ordinary-sleep command gate middleware and focused sleep-gate assertions.
- Split restart handling into request and confirmed execution paths, with stale confirmation checks against the current Telegram id.
- Added `MAP-WILLOW-002` backlog planning for a future under-bridge / watchtower integration pass around the Willow Floodplain.
- Added `WORLD-003` near-term planning for moving hunger, torches, campfires, corpse/meat decay and similar temporary lifetimes onto internal world-time hours/days.
- Reconciled planning status for shipped world-time, visibility, fire/light, `twigs`, ordinary sleep and restart-safety slices after 0.14.22.
- Clarified `PROG-002` as an umbrella learning-design item rather than a competing near-term implementation slice.
- Refreshed planning exports after adding the new planning item.

## Risks

- `/shout` without text only prompts for usage; it does not open an interactive text-entry flow yet.
- The command still follows the existing speech/action queue and proactive/session guard behavior.
- `MAP-WILLOW-002` is planning-only and does not add routes, open settlement access or alter the under-bridge location in this patch.
- The sleep command gate intentionally blocks `/sleep tutorial` while already in ordinary sleep; players should wake before entering the tutorial dream again.
- The restart confirmation is Telegram inline-button based, so stale old confirmation buttons can still be clicked, but they now fail safely if they no longer match the current player.
- Hunger is only a bridge balance fix here; the durable internal-world-time hunger and item lifetime pass remains future work under `WORLD-003`.
