# 0.14.23 - Clickable shout command polish

Date: 12026-06-01

## Summary

This tiny stabilization patch makes the `shout` command safer as a clickable Telegram command hint. The wider speech behavior itself does not change.

## Player-Facing Notes

- Clicking `/shout` without text now reaches an explicit command handler and shows the expected format instead of falling through.
- `shout <message>` and the existing Ukrainian text aliases still use the current region-wide speech action and stamina cost.

## Technical Notes

- Added a direct `bot.command("shout", ...)` route that reuses the existing shout submission path.
- Kept the slashless and Ukrainian alias parser behavior unchanged.
- Added `MAP-WILLOW-002` backlog planning for a future under-bridge / watchtower integration pass around the Willow Floodplain.
- Refreshed planning exports after adding the new planning item.

## Risks

- `/shout` without text only prompts for usage; it does not open an interactive text-entry flow yet.
- The command still follows the existing speech/action queue and proactive/session guard behavior.
- `MAP-WILLOW-002` is planning-only and does not add routes, open settlement access or alter the under-bridge location in this patch.
