# 0.14.3 - Weather MVP and light snapshot foundation - 12026-05-31

This patch gives the internal world clock its first weather state and adds a shared light snapshot helper for future visibility work.

## Added

- Added a small internal weather service with persistent weather key, intensity and next-change world minute.
- Added `/weather` and text aliases such as `weather` and `погода`.
- Added non-proactive `WorldEvent` rows when internal weather changes.
- Added a shared light snapshot helper for daypart, moon illumination, weather modifiers and local active light.
- Added focused tests for weather display, weather light modifiers and light snapshot behavior.

## Changed

- `/time` now includes weather and compact light summary lines.
- Player/admin-created ordinary campfire ashes now decay by internal world time: after two in-game days they disappear from ordinary location lists, with a near-gone state during the final two in-game hours.
- `/help`, `/commands` and input-alias documentation now list `/weather`.
- Planning docs now mark `WORLD-001-E/F` as the `0.14.3` foundation slice.

## Deferred

- No darkness or visibility reduction yet.
- No `/look` hiding/reducing location details, beings, tracks or ground objects yet.
- No weather debug setters yet; those remain `WORLD-001-G`.
- No observation-learning, `/observe`, skill sheet or hidden-presence work.

## Validation

- Ran `node scripts/test/world-time.cjs`.
- Ran `node scripts/test/campfire-decay.cjs`.
- Ran `node scripts/test/input-aliases.cjs`.
- Ran `node scripts/test/slashless-command-coverage.cjs`.

## Risks / Rollback

- Weather changes can occur on any path that advances world time, including `/time`; this is internal state/event logging only and does not push messages.
- Light snapshot labels are visible in `/time`, but normal location rendering does not consume them yet, so player expectations around darkness should stay modest until `VIS-001-*`.
- Existing extinguished handmade/debug campfires that predate this patch get their ash-decay schedule backfilled when the fire cleanup path sees them, so they are not removed immediately on deploy.
- Existing deployments already need the `WorldState` weather columns from `0.14.1`; no new migration is added here.
- Rollback: revert this patch if weather/light foundation behavior needs to be removed before visibility work continues.
