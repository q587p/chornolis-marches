# 0.14.4 - Visibility debug foundation - 12026-05-31

This patch gives the Night, Light and Firewood line its first shared visibility consumer and adds safe scribe controls for testing time, moon and weather states.

## Added

- Added `src/services/visibility.ts`, a shared helper that turns light snapshots into visibility rules for brief and detailed location views.
- Added scribe/admin `/timeDebug`, `/timeSet` and `/weatherSet` controls.
- Added `🌒 Час світу` to the scribe admin menu.
- Added focused tests for visibility decisions and time/weather debug parsing.
- Added `docs/systems/visibility.md`.

## Changed

- Brief `/look` now uses the visibility service instead of directly asking whether there is local active light.
- Clear enough daylight can reveal nearby beings, ground objects and target buttons during brief `/look`.
- Dim and dark brief views remain cautious unless local light is present; deeper atmospheric hiding/reduction is deferred.
- Production seed refresh no longer rewinds an existing `WorldState` clock during redeploy; it only creates the row when missing.
- Updated planning, admin and world-time docs for `WORLD-001-G` and `VIS-001-A`.

## Deferred

- No `/observe`, observation XP, skill sheet or hidden-presence work.
- No full darkness copy pass yet.
- No detailed `/examine` hiding or reduced track/resource displays yet.
- No new seed, migration or deploy data step.

## Risks / Rollback

- Brief `/look` now advances/reads world time through the visibility service, so time/weather can progress on a look path as well as `/time` and world tick.
- Player-facing visibility behavior changes slightly: clear/daylight may show brief nearby details where old logic required local fire/light.
- `/timeSet` and `/weatherSet` are intentionally restricted and audited, but they are still powerful world-state controls.
- Rollback: revert this patch to return brief `/look` to local-light-only reveal logic and remove the scribe debug controls.
