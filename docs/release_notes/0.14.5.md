# 0.14.5 - Darkness visibility reduction - 12026-05-31

This patch makes the shared visibility foundation visible in normal location play: dim and dark places now withhold details unless there is enough natural or local light.

## Added

- Added shared darkness copy helpers for location description, nearby presence, ground objects, resources and tracks.
- Added focused assertions for darkness visibility rules and representative darkness copy.

## Changed

- Brief and detailed location views now reduce the long description in dim/dark visibility without local light.
- Nearby names, target buttons, loose ground-object buttons, resource details and track hints now respect the shared visibility rules.
- `/track` completion checks visibility before showing track lines.
- Light from the existing light snapshot path restores normal detail when visibility becomes clear enough.
- Prepared onboarding-name summaries now render plural-form notes as Ukrainian player-facing copy instead of leaking internal English notes.
- Hunter NPCs now answer direct greetings through a delayed social response, and NPC greeting counts in ecology/status statistics count completed creature `GREET` actions.
- Scribe `/locationAll` output now sorts by region first, keeping the dream tutorial region before the rest of the world.
- Updated planning, world-time and visibility docs for `VIS-001-B/C/D/F`.
- Bumped package metadata to `0.14.5`.

## Deferred

- No `/observe`, observation XP, skill sheet or hidden-presence work.
- Feature-specific darkness copy and hidden target lookup copy remain follow-up work.
- No new seed, migration or deploy data step.

## Risks / Rollback

- Players in dim/dark conditions may see less information from `/look`, `/examine` and `/track` until they bring light or daylight improves.
- Some existing location descriptions that were always visible are now hidden by visibility rules in dark conditions, which may affect navigation feel.
- Track actions still spend stamina before reporting that the tracks cannot be read clearly.
- Direct hunter greeting reactions are queued and delayed, so busy hunters may answer after their current action rather than immediately.
- Rollback: revert this patch to return descriptions, targets, ground objects and tracks to the 0.14.4 visibility behavior.
