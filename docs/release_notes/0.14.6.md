# 0.14.6 - Boundary Mark Chancery Herald bot - 12026-06-01

This release adds the Boundary Mark Chancery / Канцелярія Межового Знаку as a separate Telegram bot runtime for news, world digests and service announcements.

The main game bot remains the production entrypoint for gameplay. The Herald has its own standalone entrypoint:

```bash
node dist/apps/heraldBot.js
```

## Added

- Added a standalone Herald Telegram bot runtime with its own token config, admin-id parsing, `/ping`, `/help`, `/whoami`, unknown-command fallback and minimal HTTP health server.
- Added latest `news.md` preview/post commands and durable `HeraldPublication` outbox storage with content-hash deduplication.
- Added a publisher loop for queued HeraldPublication rows, plus admin commands to inspect and publish pending rows.
- Added world digest preview/queue/post commands using aggregate, spoiler-safe world data.
- Added atmospheric `/info` support that turns character records into qualitative text instead of raw stat tables.
- Added historical `news.md` backfill commands for archive drip-feed posting.
- Added startup notice support for a configured admin/private chat.
- Added a `ServiceHeartbeat` table and Herald heartbeat loop so the status page can show Chancery mode, freshness, queued/published counts and stale warnings.
- Added Telegram deep-link start payloads for safe main-bot actions such as `cmd_look` and `cmd_examine`.
- Added Herald safety helpers and smoke tests for parsing, content hashes, admin/channel ids, public text guards and status helpers.

## Changed

- Split the main game bot runtime into `src/apps/gameBot.ts` while preserving `src/bot.ts` as the existing compatibility entrypoint.
- Documented standalone Render Web Service deployment for the Herald, including the build command with `prisma migrate deploy`.
- Documented future embedded Herald mode as an icebox option only; it is not enabled by this release.
- Updated the public/admin status page service section to include the main game bot, Boundary Mark Chancery, action queue and world tick overview.

## Deferred

- Embedded Herald mode inside the main Render Web Service remains a future task.
- The Herald does not yet automatically decide when a world digest is worth posting beyond the configured publisher loop and manual/admin queue commands.
- Status heartbeat is runtime-level health, not a guarantee that Telegram channel permissions are correct.

## Risks / Rollback

- Herald uses Telegram long polling. Running two active processes with the same `HERALD_BOT_TOKEN` can cause Telegram `409 Conflict` errors.
- `HeraldPublication` deduplicates by content hash; intentionally reposting identical text will need a deliberate future override path.
- The status page reports Herald heartbeat freshness only after the Herald polling runtime has started successfully.
- Rollback: revert this release to remove the standalone Herald runtime, outbox migrations, status heartbeat and Chancery docs while keeping the main game bot entrypoint on its previous `src/bot.ts` path.
