# 0.14.9 - Starter Camp Verticality - 12026-06-01

This release gives the starter border camp its first real vertical location and
moves the beginner torch source into that space.

## Added

- Added `start_border_watchtower`, a real `z = 1` location above
  `start_border_camp`.
- Added `UP` and `DOWN` exits between the starter camp and the watchtower.
- Added an inspectable `Сторожова вежа` feature at the camp.
- Added watchtower lexicon forms for feature targeting.
- Added world-seed checks for the watchtower, vertical exits and beginner torch
  placement.

## Changed

- Updated the starter camp description to mention the tower and ladder.
- Moved `start_camp_torch_stand` from the camp floor to the watchtower awning.
- Kept the closed-gate torch source unchanged until hunter routes and gate-side
  supply are audited.
- Updated world-map, fire/light, onboarding, z-level and planning docs.

## Validation

- Ran `node scripts/test/world-seed.mjs`.
- Ran `node scripts/world/render-map-ascii.mjs --write`.

## Risks / Rollback

- This branch is stacked on the 0.14.8 starter-camp clarity branch. Merge or
  rebase after 0.14.8 lands on `main`.
- Beginner torch pickup now asks the player to notice and use the upward path
  first. This is intentional, but first-session telemetry/playtesting should
  watch whether new players miss it in darkness.
- The old gate torch source remains, so there are still two authored torch
  sources until the hunter route and supply audit decides the final shape.
- Rollback: revert this patch to remove the watchtower location/exits and move
  `start_camp_torch_stand` back to `start_border_camp`. No schema migration is
  included.
