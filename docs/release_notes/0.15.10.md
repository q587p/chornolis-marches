# 0.15.10 - Movement target buttons and animal escalation - 12026-06-02

This patch finishes two small follow-ups from the movement/speech polish line: safer target buttons for coalesced movement messages and a first repeated-wolf-speech escalation.

## Summary

- Coalesced movement notifications now check creature state immediately before sending and add target buttons only for living, visible, non-animal creatures still in the target location.
- Stale movement target buttons are still cleared when a coalesced movement message flushes, so quick arrival/departure sequences should leave fewer misleading buttons.
- Repeated directed speech toward the same wolf within a short runtime window can escalate from a warning growl to a local danger marker.
- Added `ANIMAL_SPEECH_PROVOCATION_WINDOW_MS` and `WOLF_SPEECH_PROVOCATION_THRESHOLD` tuning knobs.
- Updated movement notification and animal speech focused tests, planning docs, public news and changelog.

## Risks / Rollback

- This branch is intentionally stacked on the 0.15.9 movement/speech PR until that PR merges; rebase/update it onto `main` after 0.15.9 lands.
- The movement coalescer remains an in-memory debounce. If the process restarts during the window, pending flavor lines may be dropped; world state and tracks are unaffected.
- Movement target buttons are DB-backed at flush time, but they only cover non-animal visible creatures. Animals remain reachable through the current location/target surfaces instead of movement-arrival buttons.
- Wolf escalation is also in-memory and does not directly damage players. Richer predator-vs-player combat remains deferred to the broader combat/predator behavior work.
- Rollback: revert the notification-buffer/action-completion helper changes, docs and version bump if chat cadence or wolf reactions feel worse in live play.
