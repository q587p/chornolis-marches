# 0.15.11 - Camp cat cache presence and mouse priority - 12026-06-02

This patch keeps the early camp-cat line small: the cat feels more tied to the starter cache and local mice without becoming a pet, a loot source or a full predator system.

- Added an optional Кіт-бережник presence line to starter shared-cache inspection when the cat is visibly in the same location.
- Kept cache use unchanged: the cat does not own, guard, reward, punish or block starter-cache supplies.
- Added a first camp-local mouse-priority behavior: if live visible mice are in the cat's current camp/watchtower location, the cat chooses its mouse-listening watch posture before ordinary up/down movement.
- Updated camp-cat, beginner-cache, ecology, onboarding and planning docs.

## Risks / Rollback

- This branch is intentionally stacked on the 0.15.10 PR until that PR merges; rebase/update it onto `main` after 0.15.10 lands.
- The cache cat line appears only when the cat is currently visible in the same location. If the cat is downstairs while the cache is upstairs, the cache text remains unchanged.
