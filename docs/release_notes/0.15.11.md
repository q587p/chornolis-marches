# 0.15.11 - Camp cat raw-meat give bridge - 12026-06-02

This patch adds the first narrow `give` bridge without turning it into a full exchange system: raw meat can be offered to the visible camp spirit cat through the same canonical path from text aliases and target UI. It also includes small UX and reliability polish that belongs with the same handoff/communication slice.

## Risks / Rollback

- This is intentionally a tiny bridge, not full barter: only one raw-meat unit to the visible camp spirit cat is supported.
- Shared-cache contribution buttons are denser now; long resource names may wrap on narrow Telegram clients, but callback paths are unchanged.
- Blocked Telegram users now simply miss private target deliveries for greetings, whispers, creature replies or social signals; the actor response, room notification, action completion and non-blocked deliveries continue.
- Completed freshening now only removes stale source buttons instead of rewriting the old source message text; old Telegram messages may still keep stale buttons if Telegram refuses a markup edit.
- The action is immediate and records a zero-duration `WAIT` note instead of adding a new Prisma `WorldActionType`; broader queued transfer can be added when exchange expands.
- The target button only appears after inspecting/focusing the cat and only if the player currently has raw meat, so live UX should be watched for discoverability.
- Rollback: revert this patch to remove the give bridge, cat feed affordance, shared-cache button compaction, freshening source-message cleanup, blocked-user send hardening, yell prompt polish, docs/news and version bump. No migration or seed refresh is required.
