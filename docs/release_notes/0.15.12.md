# 0.15.12 - Ukrainian give parser and rest recovery polish - 12026-06-02

This patch polishes the first generic `give` bridge from `0.15.11` and tunes active rest feedback: the existing raw-meat-to-camp-spirit-cat transfer now accepts common Ukrainian item and target forms instead of requiring one rigid phrase, while character recovery hints make stamina recovery and campfire rest value easier to read.

## Changes

- Added the first `GIVE-002` parser slice for the already-supported `raw_meat` to camp spirit cat transfer.
- Canonicalized raw-meat item variants such as `сире м'ясо`, `сирого м'яса`, `м'яса` and missing-apostrophe `мясо` to the same supported `raw_meat` give item in this path.
- Canonicalized camp cat target forms such as `коту`, `котові`, `кіт`, `бережнику`, `бережникові` and hyphenated `котові-бережнику` to the same visible target search.
- Kept `/give`, slashless `give`, Ukrainian `дати`, `/feed_raw_meat`, `/feed_raw_meet` and `/feed_cat` on one canonical action path.
- Updated `/help`, input-alias docs, camp spirit cat docs and planning notes to mark this as the first implemented `GIVE-002` slice.
- Increased active-rest stamina recovery from `13` to `26` stamina per rest interval.
- Active campfires now provide a minimum `x3` stamina-rest speed multiplier. Stronger authored location multipliers still take precedence.
- Character-card recovery copy now separates stamina estimates from HP estimates and shows `without rest`, `with rest` and `near campfire` stamina timing instead of embedding `/rest` in the text.
- Added a runtime first-use mushroom warning so the first mushroom eaten after this release cautions that mushrooms can be poisonous and may be worth saving for future elixirs or tinctures.
- Recorded follow-up planning for persisted food-use statistics instead of trying to infer older eating history retroactively.

## Risks

- This is intentionally narrow: only the current raw-meat-to-camp-spirit-cat transfer is canonicalized. General player/NPC gifts, item instance selection, multiple-target ambiguity UI and barter remain future work.
- The parser accepts `м'ясо` as raw meat only inside this `give` path. Inventory eating/cooking semantics still keep their own resource resolution.
- Rest now clears stamina debt faster, especially near active campfires. Watch live balance around campfire-heavy areas and stamina-heavy actions.
- Recovery estimates are still real-time approximations based on current tick settings; future world-time presentation can make these more diegetic.
- The mushroom warning is runtime-only and first-use-from-this-release, not historical. Process restarts can show it again until a future persisted food-use/statistics layer exists.
- This branch is stacked on the `0.15.11` review branch until that PR merges; rebase onto `main` after `0.15.11` lands.

## Rollback

- Rollback: revert this patch to remove the Ukrainian give parser polish, rest recovery tuning, mushroom warning, docs/news and version bump. No migration or seed refresh is required.
