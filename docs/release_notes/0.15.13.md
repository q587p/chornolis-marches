# 0.15.13 - Strange totems and dismantle action - 12026-06-02

This patch adds the first small Strange Totem wilderness loop. Dry luka and riverbank locations can now contain suspicious dry landmarks that can be inspected, dismantled for `twigs`, age out with world time and occasionally appear as ambient traces when the world is quiet enough.

## Changes

- Added Strange Totem `LANDMARK` features to dry luka and riverbank seed data.
- Added queued `DISMANTLE_TOTEM` support with stamina cost, action priority, posture/sleep guards and action-completion handling.
- Added a `Розібрати` feature button plus `/dismantle_totem`, slashless `dismantle totem`, English MUD-style aliases and Ukrainian aliases such as `розібрати тотем`.
- Added direct `/start cmd_dismantle_totem` routing so public command links can open the same action path.
- Added Strange Totem inspection text that separates fresh, old and nearly-gone states.
- Strange Totems now age by world time: after six in-game days they are old, can shed one `twigs` bundle once, and after seven in-game days they deactivate.
- Added a daily ambient spawn attempt for dry luka and riverbank locations with region caps, no-active-player and no-visible-non-animal-presence safety checks.
- Ambient totem spawns now leave a suspicious track toward a random visible exit.
- Updated `/help`, `/commands`, location-feature docs and fire/light docs for the new totem dismantle source of `twigs`.
- Added focused regression coverage for totem seed data, age states, rewards, aliases and action-cost/title wiring.

## Risks

- This release adds a Prisma enum migration for `DISMANTLE_TOTEM`; deploys must run migrations before the bot processes the new action type.
- Ambient spawn tuning is intentionally conservative, but `WORLD_STRANGE_TOTEM_DAILY_CHANCE`, region caps and spawn safety checks may need live adjustment if dry luka or riverbank locations feel too busy.
- Suspicious tracks use a hidden technical `strange_totem_trace` creature actor to satisfy the current `WorldTrack` relation. Player-facing text should not expose that implementation detail, but future track modeling should remove the fake actor.
- Totem schedules are stored in feature `data`; reset/reseed flows can reset authored feature data until a richer runtime feature-state model exists.
- Daily spawn attempts are recorded from this release forward only; older world periods are not backfilled.

## Rollback

- Rollback: revert this patch, including the `DISMANTLE_TOTEM` enum migration, Strange Totem service, seed features, aliases, docs/news and version bump. If the migration has already been deployed, keep the enum value in the database and remove only the runtime use until a follow-up cleanup is planned.
