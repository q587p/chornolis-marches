# 0.15.14 - Herald Manual Archive Controls - 12026-06-02

## Summary

- Added Herald admin controls for manual archive recovery from deployed `news.md`: `/news_archive_list`, `/news_archive_preview <index>`, `/news_archive_post <index>` and `/news_archive_reload`.
- Added `HERALD_PUBLICATIONS_PAUSED` startup safety so the standalone Herald publisher can wake on Render without immediately draining a large pending outbox.
- Kept `/pause_publications`, `/resume_publications`, `/cancel_pending_publications` and `/backfill_news_cancel` aligned with visible queue counts.
- Documented the Render operator flow for pausing or canceling a large backfill and then publishing old archive entries one at a time.

## Public News

Added a short atmospheric public news entry about the Boundary Mark Chancery organizing old news scrolls. It avoids admin command names and internal operator details because the gameplay rules themselves did not change.

## Validation

- `node scripts/test/herald-smoke.cjs`
- `npm test`
- `npm run build`
- `git diff --check`

## Risks

- Manual archive posting reads the deployed `news.md`; changing archive text on Render still requires commit, push and redeploy before the commands see it.
- `/news_archive_post <index>` posts immediately to `HERALD_CHANNEL_ID`; it refuses content already published by `contentHash`, but there is no force mode yet for intentional reposting.
- `HERALD_PUBLICATIONS_PAUSED=true` pauses the publisher at startup, but an already in-flight Telegram send can still finish before a runtime pause is observed.
