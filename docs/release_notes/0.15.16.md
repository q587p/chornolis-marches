# 0.15.16 - Money and Low-Risk Loot MVP

Date: 12026-06-02

## Summary

0.15.16 adds the first bounded money/low-risk loot slice: `shah` and `hryvnia` become named resources, a few starter-adjacent `shah` finds can be noticed and picked up, `/me` can describe carried money, the beginner cache gets a separate money box, and successful waking-world gathering can very rarely turn up one `shah`.

This is deliberately not a shop, vendor, price, faction or theft system.

## Changes

- Added `shah` and `hryvnia` to world resource types, lexicon data and grammar helpers.
- Added shared money text helpers for correct `шаг` / `ґривня` pluralization.
- Updated `/me` to show a money line, including `Гроші: немає` when the player carries no money.
- Added `shah` and `hryvnia` to visible ground-resource pickup support and item/resource text aliases.
- Seeded six small starter-adjacent `shah` finds and kept tutorial-dream money and broad starter `hryvnia` finds out of scope.
- Added a separate beginner shared-cache money box with money-specific take/contribute buttons.
- Kept hidden beginner-cache restock from creating money.
- Added a rare waking-world successful-gather `shah` bonus through `src/services/smallLoot.ts`, with tutorial/dream locations excluded.
- Added focused tests in `scripts/test/money-loot.cjs` and extended seed checks.
- Added `docs/systems/money_and_low_risk_loot.md`, `MONEY-001` and updated `LOOT-001` planning notes.
- Regenerated planning exports.

## Public News

Added a public `news.md` entry about first small coins, nearby quiet finds, `/me` money display, visible pickup through `/examine` / `/get_all`, the shared-cache money box and rare waking-world gathering coin finds.

## Risks

- The beginner cache stores coin stock in feature data for this MVP; a later durable item/stock layer may need migration rules if shared-cache stewardship grows.
- The rare gather bonus calls the money-resource upsert helper defensively so live deployments can grant coins even if seed has not been rerun, but long-term resource definitions should still come from committed seed data.
- Starter `shah` finds are static authored `ResourceNode`s; without item lifetime or replenishment rules they can be exhausted or reset only through existing seed/reset flows.
- `/get_all` can pick up visible money together with other visible ground resources, so future queued-pickup and stamina tuning should include money piles.
- `hryvnia` is defined and movable through shared paths, but broad `hryvnia` discovery remains intentionally unseeded until a later authored use exists.
