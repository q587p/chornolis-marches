# 0.15.17 - Observation Cues and Quiet Omens

Date: 12026-06-02

## Summary

0.15.17 makes full location examination a little more informative before the larger observation-learning system lands. `/examine` can now add qualitative danger atmosphere, empty-find copy for calm/starter-adjacent searches, and richer conditional owl-sign text.

This is a copy-and-helper slice, not a new `/observe` command, skill model or combat/fear system.

## Changes

- Added `src/services/locationDanger.ts` for qualitative effective-danger bands used by full location examination.
- Added `src/services/locationExamineCues.ts` for empty-find copy when no visible small finds, gatherable resources, corpses or tracks are present.
- Wired danger and empty-find cues into `renderLocationDetails` only; ordinary `/look` stays compact.
- Extended owl-sign inspection copy for dusk/night and dawn while keeping day inspection quiet.
- Changed `/adminHelp` to show a compact section index with focused command groups, avoiding Telegram's single-message length limit for the full scribe command list.
- Kept tutorial and lucid dream locations on stable dream light by default, so waking-world daypart/weather darkness no longer hides tutorial descriptions, targets, tracks or ground objects.
- Added concise full-inspection summaries for tutorial action-ladder features such as `Мох уважного погляду`, and split the signs lesson into the three visible signs described in prose so `/examine` in those dream rooms exposes a second layer instead of only repeating the feature label.
- Fixed game deep-link command payload handling so Herald/site links such as `/start cmd_look` and `/start cmd_examine` dispatch to the intended public command when the character is in the waking world.
- Polished Herald world digest formatting so preview/channel text has one `Запис із краю Чорнолісу` heading, a short in-world date line and clearer paragraph spacing.
- Added `scripts/test/observation-cues.cjs` and included it in `npm test`.
- Added high-priority `ONB-009` planning for cleaning unrelated ground items/resources out of the tutorial dream.
- Updated `OBS-003`, `OWL-004`, `LOOT-001`, system docs and planning exports.

## Public News

Added a public `news.md` entry about attentive examination sensing local danger, empty searches feeling local/temporary, owl signs carrying slightly richer time-sensitive text, and tutorial dreams staying readable despite waking-world darkness.

## Risks

- Danger cues use a lightweight effective-danger approximation in the examination renderer. It intentionally mirrors existing pressure concepts, but future observation skill work may want a shared persisted/effect service.
- Empty-find copy appears when full examination sees no visible finds/resources/corpses/tracks. This should make barren moments feel better, but live play may show locations where the line feels too chatty.
- Owl-sign copy is still feature-based; visible owl creature examination may need its own richer conditional copy in a later observation pass.
- Dream light is intentionally broad for dream-keyed or low dream-layer locations. If future non-dream underground content uses very low `z` layers, it should opt into ordinary darkness explicitly or avoid the reserved dream layer.
