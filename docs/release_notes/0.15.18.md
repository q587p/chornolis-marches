# 0.15.18 - Tutorial Cleanup and Calendar Split

Date: 12026-06-03

## Summary

0.15.18 keeps the tutorial dream focused after the wider loot/ground-item work and splits the overloaded world-time surface into three clearer player commands: `/time`, `/calendar` and `/weather`.

This is a UX and command-surface slice. It does not add deeper calendar simulation, seasons, sacred days or new dream rooms.

- Added `scripts/test/tutorial-ground-items.cjs` and included it in `npm test`.

## Risks

- Tutorial dream cleanup is currently a visibility/pickup-boundary rule, not a database deletion or persisted origin-marking system. Unrelated dream resources may still exist in the database, but they should not appear or be picked through normal tutorial ground-item paths.
- `/time` is intentionally shorter now. Players who were used to getting the full year/circle/moon bundle from `/time` need `/calendar` instead.
- `/calendar` naming is the obvious first choice; if play text later suggests a more diegetic name, keep `/calendar` as a compatibility command and add the better label as an alias.
