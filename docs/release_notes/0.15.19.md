# 0.15.19 - Apiary And Bumblebee Hazard MVP

Date: 12026-06-03

## Summary

0.15.19 adds the first apiary / old log hive to the dry meadow and gives it a small living-world hazard: rare passive bumblebee stings while players move, inspect or wait nearby.

This is an ambient hazard slice, not honey harvesting or a bear loop.

## Changes

- Added `meadow_old_log_apiary_12_02` as an inspectable `LANDMARK` feature in `meadow_12_02`.
- Added apiary metadata for aura radius, passive sting chance/damage, night sleeping, future honey/wax rewards and future bear hooks.
- Added `src/services/apiaryHazards.ts` with pure helpers and runtime handling for passive bumblebee stings.
- Hooked passive checks into successful player movement, queued full location look, direct full location inspection, feature inspection and waiting.
- Stored sting cooldown memory through `WorldEvent` records rather than adding a new schema table.
- Added `scripts/test/apiary-hazards.cjs` and seed assertions for the apiary feature.
- Added `docs/systems/apiary-and-bumblebee-hazards.md`, `docs/planning/apiary-implementation-plan.md` and APIARY planning items.

## Risks

- Passive sting notification cadence depends on chance plus a WorldEvent cooldown. The cooldown is intentionally conservative, but live chat volume should still be watched after deploy.
- The hazard can lower HP, though the MVP clamps passive damage so it cannot kill. Future deliberate hive robbery may need a stricter recovery/knockout design before it can be harsher.
- Honey and beeswax are only future hooks in this release. Players can inspect the apiary and experience the hazard, but cannot harvest it yet.
- Cooldown memory uses `WorldEvent` rather than a dedicated per-player hazard table. If events are pruned aggressively, cooldown history can disappear.
