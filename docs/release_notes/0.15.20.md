# 0.15.20 - Apiary Honey And Wax Raid

Date: 12026-06-03

## Summary

0.15.20 turns the old log apiary from a passive hazard into a small risky resource point: players can deliberately disturb the hive for honey and sometimes beeswax.

This is still a narrow apiary slice, not a bear loop, shop/economy pass, crafting pass or full swarm creature system.

## Changes

- Added `RAID_APIARY` as a queued world action with action title, stamina cost, posture/sleep integration and completion handling.
- Added `/gather_honey`, `/gather_beeswax`, `gather honey`, `взяти мед`, `зібрати мед`, `обібрати вулик`, `пограбувати бортю` and `добути віск` routes.
- Added a feature button on apiary inspection for risky honey collection.
- Added `honey` and `beeswax` resource types and lexicon forms.
- Added inventory lookup/drop/inspect aliases for honey and beeswax.
- Added scribe `/addItem` support and admin menu buttons for honey and beeswax.
- Added safe deep-link payloads for `/gather_honey` and `/gather_beeswax`.
- Added tests for apiary raid helpers, action cost/title, posture blocking, input aliases, admin item parsing, Herald links and seed metadata.
- Updated apiary docs, planning notes, changelog and public news.

## Public News

Added a public `news.md` entry explaining that the old log apiary can now be disturbed for honey, that beeswax may appear, and that deliberate robbery wakes the swarm even at night.

## Risks

- `RAID_APIARY` adds a PostgreSQL enum value. Rollback should usually mean reverting code while leaving the unused enum value in production, unless a deliberate enum rebuild migration is prepared.
- Raid cooldown memory uses `WorldEvent` records with an `Apiary raid` title and `apiaryKey=...` marker. If old world events are pruned aggressively, the hive may become available earlier than intended.
- Honey and beeswax are inventory resources in this slice, but not yet fully usable food, remedies, offerings, crafting materials or trade goods. That is intentional follow-up scope.
- The no-target typed commands pick the active apiary at the current location. If multiple active apiaries later share a location, feature buttons or target arguments should become more precise.
