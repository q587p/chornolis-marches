# 0.15.21 - Attentive Presence Learning Bridge

Date: 12026-06-03

## Summary

0.15.21 adds the first small persistent learning/progress foundation and connects one narrow observation-learning path to it. Characters can now begin accumulating hidden canonical progress from attentive presence around visible gathering of herbs, berries and mushrooms, while ordinary players still see only diegetic milestone text and no raw numbers.

This is not a broad skill sheet, not a full `/observe` command, and not a general resource-learning system.

## Changes

- Added the `CharacterLearningProgress` Prisma model for player + skill + source + context progress rows.
- Added `src/services/learning.ts` with reusable `recordLearningProgress`, `getLearningProgress` and milestone helper logic for future observation, tracking and practice slices.
- Added a narrow gathering observation bridge through existing `look`, `examine`, character inspection and witnessed local action surfaces.
- Limited canonical gathering observation progress to herbs, berries and mushrooms.
- Preserved hidden legacy `WorldEvent` observation markers/milestones while keeping unsupported resource sources out of canonical progress.
- Added focused helper tests for learning progress and gathering observation scope.
- Updated planning/progression documentation so `OBS-001` and `TRACK-LEARN-001` can build on the shared learning foundation.

## Public News

Added a public `news.md` entry explaining that attention now has its first hidden memory layer: careful looking and inspection near visible gathering can begin teaching without exposing raw progress numbers or adding a new command.

## Validation

- `npm run planning:export`

- `git diff --check`

## Risks

- This release adds a Prisma migration for `CharacterLearningProgress`; deploys need the normal migration step before the new code writes learning rows.
- Gathering observation still keeps legacy hidden `WorldEvent` markers for cadence and milestone compatibility. Unsupported resources such as honey and beeswax do not create canonical learning progress, but they may still leave non-canonical observation markers if the old cadence path sees them.
- No ordinary player-facing skill UI is added. Scribes/admins may eventually need technical views for learning rows, but that is intentionally outside this slice.
- The first bridge is intentionally narrow. Future track-reading, herbalist, fisher, hunter and apprenticeship moments should reuse the helper instead of expanding this patch into a broad skill system.
