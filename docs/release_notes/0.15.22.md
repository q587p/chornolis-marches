# 0.15.22 - Learning Foundation Stabilization

Date: 12026-06-03

## Summary

0.15.22 is a small stabilization pass for the 0.15.21 attentive-presence learning foundation. It keeps the scope narrow: no `/observe` command, no follow intent, no skill UI and no skill effects.

The main runtime changes are a safer gathering-observation source scan and an apiary cooldown fix. A recently observed or unsupported gather-like source should no longer block a still-valid herbs, berries or mushrooms observation in the same short attention window, and old log apiary raid cooldown now follows in-world time markers instead of effectively waiting on wall-clock time.

## Changes

- Updated `recordGatheringObservation(...)` to scan a small set of recent gathering source events instead of only the newest one.
- Kept canonical learning progress limited to herbs, berries and mushrooms.
- Preserved the legacy hidden observation marker path for unsupported sources without making them canonical learning progress.
- Added a regression test for unsupported recent sources no longer masking older supported gathering sources.
- Renamed the public old-log-apiary command hints to `/search_honey` and `/search_beeswax`, because these are risky feature interactions rather than ordinary location-resource gathering. The old `/gather_honey` and `/gather_beeswax` names remain legacy compatibility routes.
- Fixed old log apiary raid cooldowns to store and check `absoluteMinute` in the `WorldEvent` marker, so honey collection can become available again after the intended in-world cooldown instead of feeling one-shot in live play.
- Shortened the authored old log apiary raid cooldown to roughly six in-game hours.
- Documented and kept the old log apiary passive center sting tuning at `2-3` HP damage, while adjacent aura stings remain lighter.
- Updated progression and next-planning docs to mark this as stabilization before follow intent, track-learning or skill-effect work.
- Added public news and changelog entries for the stabilization slice and updated apiary command hints.

## Public News

Added a short public `news.md` entry explaining that attention around recent gathering events is a little steadier and that old log apiaries should become approachable again after time passes, while pointing players to `/search_honey` and `/search_beeswax`.

## Risks

- Gathering observation still uses a short event window. If locations become very noisy, the scan limit may need tuning, but this patch avoids broad observation refactors.
- Unsupported sources such as honey still can create legacy observation markers for cadence/dedupe compatibility, but they intentionally do not create canonical learning progress.
- Apiary cooldowns now depend on `absoluteMinute` written to new `WorldEvent` descriptions. Older live raid/sting events without the marker are treated as expired, which may allow one earlier repeat harvest after deploy but avoids stale real-time blocks.
- The public news mentions `/look`, `/examine`, `/search_honey` and `/search_beeswax`; all should stay registered and covered by clickable-command tests.
