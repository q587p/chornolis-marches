# 0.15.23 - Progression Stabilization And Gathering Skill Tuning

Date: 12026-06-03

## Summary

0.15.23 turns the learning foundation into something slightly more inspectable
and useful without opening a broad skill system. It adds a technical 0..5 level
policy, a scribe-only `/learning` view and the first narrow mechanical effect:
canonical gathering progress for herbs, berries and mushrooms can now make that
same kind of gathering a little steadier and slightly less stamina-heavy.

There is still no public `/skills`, no `/observe` command and no broad skill
tree.

## Changes

- Added a shared 0..5 learning level policy in `src/services/learning.ts`, with
  total-progress thresholds at `0`, `3`, `8`, `18`, `35` and `60`.
- Added technical learning level labels for scribe/debug views.
- Added the scribe-only `/learning [#id|name|username]` command to inspect one
  character's stored `CharacterLearningProgress` rows.
- Added a bounded gathering skill effect for herbs, berries and mushrooms:
  higher matching gathering progress can slightly improve success chance and
  reduce stamina cost, with hard caps and a hard stamina-cost floor.
- Kept honey, beeswax, twigs, money, loot-like sources and missing resource
  metadata outside canonical gathering skill effects.
- Added focused helper coverage for learning thresholds and supported/unsupported
  gathering skill effects.
- Updated progression, planning and admin-command documentation for the new
  technical surface and the remaining follow-up scope.

## Public News

Added a public `news.md` entry describing the visible player-facing effect:
careful practice around herbs, berries and mushrooms can make those small
searches a little steadier. The scribe-only `/learning` command is intentionally
not mentioned in public news.

## Validation

- `node scripts/test/learning.cjs`
- `node scripts/test/gathering-learning.cjs`
- `npm test`
- `npm run build`
- `git diff --check`

## Risks

- The first skill effect is intentionally small, but gathering stamina and
  success balance may still need live tuning after players accumulate progress.
- `/learning` exposes raw stored progress only to scribes/admins; access control
  should stay guarded by the same scribe checks as other technical commands.
- Existing legacy learning messages for attack, food and older gathering paths
  remain partly event-based. Later slices should migrate them carefully instead
  of rewriting all learning behavior at once.
