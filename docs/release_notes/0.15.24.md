# 0.15.24 — Spirit Call auto visibility

This patch rethemes the existing player automatic-action helper as `Поклик духа` in player-facing surfaces while keeping the old internal `auto` implementation and legacy commands stable.

## Player-facing changes

- `/spirit`, `/dukh` and `/poklyk` now start the same simple automatic-action helper that `/auto` already used.
- `/spirit_stop` and `/dukh_stop` stop it, alongside existing `/auto_stop` compatibility.
- Character-card buttons and confirmation copy now speak about calling or releasing the spirit instead of turning auto mode on/off.
- Nearby target lists and player inspection can show a subtle hint when a character is guided by the spirit call.
- Full scribe/admin player detail cards now include a compact `Навчання` summary before statistics, with a `/learning #id` pointer for the complete raw progress view.

## Technical notes

- Automatically created player actions now carry a `source: "spirit_call"` / `spiritCall: true` payload marker.
- Visible running/queued action labels use that marker, so manual player actions while the helper is enabled are not mislabeled as spirit-guided.
- Internal `auto:*` action notes, scheduler naming and old `/auto` command compatibility are intentionally preserved.
- The rejected `Провід` label is documented as not being used for this feature.

## Tests

- Added focused coverage for spirit-call payload helpers.
- Extended alias and help-copy tests for `/spirit`, `/dukh`, `/poklyk`, `/spirit_stop`, `/dukh_stop`, `поклик духа`, `подякувати духу` and related Ukrainian text aliases.
- Added focused coverage for compact admin learning-summary formatting and overflow protection.

## Risks

- This is a naming/visibility layer over the existing helper, not a new AI, follower, hunting mode or profession automation system.
- Older docs and historical changelog/news entries still mention `auto` for past releases. Current player-facing surfaces should prefer `Поклик духа`, while technical docs may still use `auto` where it names internals.
- Existing `/auto` and `/auto_stop` links remain valid; removing them would be a separate compatibility decision.
- The admin player card still uses Telegram message truncation as a final guard. The embedded learning block is intentionally compact and points scribes to `/learning #id` for longer raw progress lists.
