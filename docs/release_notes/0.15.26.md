# 0.15.26 — Herbalist cellar supply-run MVP

Date: 12026-06-03

## Summary

- Added the first small visible herbalist supply-run routine using the starter cellar as a staging/rest point.
- Moved herbalist tick behavior into `src/services/npcHerbalist.ts`, with `worldTick.ts` only dispatching to the service.
- Let herbalists occasionally stage through the cellar/watchtower, symbolically check torch-light preparation, gather herbs/berries/mushrooms through existing `GATHER_SPECIFIC` creature actions, then return to the cellar.
- Kept the cellar shelf no-loot and narrative-only while marking it as future herbalist deposit staging.
- Added `Сухі лежанки` as an inspectable cellar rest landmark without adding a new sleep or comfort system.

## Validation

- Added `scripts/test/npc-herbalist.cjs` for supply-run cadence, supported gather resources, route helper output and symbolic torch-preparation text.
- Extended `scripts/test/world-seed.mjs` for the new cellar no-loot/staging feature flags.

## Risks / Watchpoints

- The supply-run uses `WorldEvent` markers and `currentAction` text as a lightweight route state. This avoids a migration, but old event cleanup could forget route cadence or an active stage.
- Torch preparation is symbolic in this slice. Herbalists do not consume, carry, burn or relight real torch inventory yet.
- A route currently targets the nearby `meadow_16_05` resource location. If resource tuning or map topology changes, the target may need a small authored adjustment.
- This is not follow intent, auto-follow, a profession system, shared storage or a container. Players can manually watch the routine through existing location/inspection surfaces.
