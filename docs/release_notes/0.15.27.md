# 0.15.27 — Darkness-aware feature inspection

Date: 12026-06-03

## Summary

- Fixed dark-location `/examine` rendering so location features no longer leak authored detail summaries without valid light.
- Added a dark-safe direct feature inspection fallback that shows only outlines and asks for a lit torch or another light source before small marks, writing or route hints can be read.
- Preserved the previous feature detail behavior when daylight, a lit torch, active campfire or another local light source makes details visible.
- Split the starter camp vertical stack into the dedicated `starter_camp` / `Межовий табір` region so future beginner-infrastructure rules can target camp, watchtower and cellar together.
- Kept starter infrastructure no-loot/no-natural-resource by seed validation and region-based regeneration guards, while allowing the camp spirit cat to include the cellar in its camp bounds.
- Removed `/call_scribes` from the short Telegram command menu; the command itself remains available through help/fallback text when manual Scribe return help is actually relevant.
- Added a hidden, one-way spoken-word passage from the starter cellar to the under-bridge. It is hinted only through cellar feature inspection and is intentionally absent from ordinary exits, buttons, public news and map docs.
- Added safe Herald/site deep-link rendering and `/start cmd_*` routing for public movement commands, including `/up`, `/down`, `/north`, `/south`, `/west`, `/east`, `/inside` and `/outside`.

## Validation

- Extended `scripts/test/tree-features.cjs` with dark/lit feature summary regression coverage.
- Extended world seed and camp-cat checks for the `starter_camp` region, no-resource starter infrastructure and cellar cat bounds.
- Extended Telegram command-menu coverage so `/call_scribes` stays out of the short public menu.
- Added focused water-word passage coverage for phrase normalization, location/state guards, hidden-exit invariants, feature hints, WorldEvent metadata and public-news spoiler checks.
- Extended Herald and start-payload smoke coverage for safe movement command deep links.

## Risks / Watchpoints

- Direct inspection in darkness intentionally limits feature action buttons to safe navigation/back affordances. If a future feature needs a specific dark-safe action, add it explicitly instead of letting all feature actions bypass the visibility gate.
- Feature names can still be visible as silhouettes in dark location views, so players can orient themselves; this patch hides summaries and full descriptions, not every landmark label.
- The new `starter_camp` region is intentionally narrow. Future beginner helpers should use the region where DB context is available, but still avoid treating the whole riverbank as starter infrastructure.
- The water-word passage is intentionally one-way and speech-triggered. Future herbalists may use it for observation learning, but this PR does not add NPC route usage or a return route.
