# 0.15.27 — Darkness-aware feature inspection

Date: 12026-06-03

## Summary

- Fixed dark-location `/examine` rendering so location features no longer leak authored detail summaries without valid light.
- Added a dark-safe direct feature inspection fallback that shows only outlines and asks for a lit torch or another light source before small marks, writing or route hints can be read.
- Preserved the previous feature detail behavior when daylight, a lit torch, active campfire or another local light source makes details visible.

## Validation

- Extended `scripts/test/tree-features.cjs` with dark/lit feature summary regression coverage.

## Risks / Watchpoints

- Direct inspection in darkness intentionally limits feature action buttons to safe navigation/back affordances. If a future feature needs a specific dark-safe action, add it explicitly instead of letting all feature actions bypass the visibility gate.
- Feature names can still be visible as silhouettes in dark location views, so players can orient themselves; this patch hides summaries and full descriptions, not every landmark label.
