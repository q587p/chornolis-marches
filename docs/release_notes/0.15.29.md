# 0.15.29 - Follow intent foundation

Date: 12026-06-04

## Summary

This release adds the first narrow follow-intent foundation. A player can mark one visible local player, creature or local character as the being whose movement they are trying to keep in mind.

This is deliberately not auto-follow, not party/group movement, not a companion system and not a learning grant by itself.

## Player-Facing Changes

- Added `/follow <target>` for visible local targets.
- Added `/unfollow` to clear the current follow intent.
- Added Ukrainian and MUD-style aliases such as `—Ā–Ľ—Ė–ī—É–≤–į—ā–ł –∑–į –∑–Ĺ–į—Ö–į—Ä–Ķ–ľ`, `–Ļ—ā–ł –∑–į –Ě–Ķ—Ā—ā–ĺ—Ä–ĺ–ľ`, `—ā—Ä–ł–ľ–į—ā–ł—Ā—Ź –∑–į –õ—É–ļ–į–Ĺ–į`, `—Ā—ā–Ķ–∂–ł—ā–ł –∑–į –ľ–ł—ą–Ķ—é`, `follow herbalist` and `–≤—Ė–ī—Ā—ā–į—ā–ł`.
- `/me` now shows a compact `–ß—É–∂–ł–Ļ —Ā–Ľ—Ė–ī: <name>.` line when a follow intent is active.

## Technical Notes

- Added `PlayerFollowIntent` with one active intent per player.
- Added `src/services/following.ts` helpers for future observation and track-learning slices.
- Follow intent stores a last-known target label and last-seen location, but ordinary players only see the label.
- Target selection reuses visible local text targets, so hidden and absent targets cannot be followed.

## Out Of Scope

- No automatic movement.
- No group/party acceptance.
- No companion ownership.
- No combat assist.
- No track-learning progress yet.
- No new herbalist route behavior.
