# 0.15.30 - Follow-aware track memory MVP

Date: 12026-06-04

## Summary

This release makes follow intent useful for the first narrow route-memory moment. If a player is following a visible local being and that target leaves through an ordinary visible exit, the follower can receive a short personal hint about the direction.

This remains attention context, not follow movement. The follower is never moved, queued, teleported or grouped by this system.

## Player-Facing Changes

- A followed visible target moving through an ordinary visible exit can produce a personal hint such as `Ви трималися чужого сліду: ... рушає на північ.`
- If darkness prevents reading the movement clearly, the hint does not reveal the direction.
- `/track` without a target now prioritizes fresh tracks that match the current follow intent and labels them as `чужий слід`.
- Hidden water-word passage movement can be remembered only as a non-directional disappearance by word/sign, not as a repeatable route or visible exit.

## Technical Notes

- Added `src/services/followRouteMemory.ts` as a narrow service around follow-intent target matching, route-memory text, event descriptions and track prioritization.
- Visible player and creature MOVE completion now calls the route-memory service after ordinary track creation.
- Player and herbalist water-word passage paths call the hidden-route memory helper without creating ordinary direction memory.
- Clear witnessed followed movement records a tiny silent `tracking` / `observation` / `followed_movement` learning row through the existing learning service.
- No Prisma migration was added for this slice; `PlayerFollowIntent`, `WorldTrack`, `WorldEvent` and `CharacterLearningProgress` are reused.

## Out Of Scope

- No automatic movement.
- No route replay.
- No group/party/travel membership.
- No companion or pet behavior.
- No hidden passage auto-repeat.
- No broad tracking skill effects or public skill UI.
- No NPC schedule changes or new map content.

## Risks / Watchpoints

- Route-memory notifications are proactive personal messages. They are movement-event scoped, but live play should still watch whether busy shared rooms feel too talkative.
- The system records tiny silent tracking observation progress on clear followed movement. It does not expose progress or change tracking modifiers yet.
- Hidden water-word usage records only a non-directional note. Future follow movement must still require learned route knowledge or an independent trigger.
