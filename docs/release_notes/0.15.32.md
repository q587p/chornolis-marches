# 0.15.32 - Try to keep following via remembered route

Date: 12026-06-04

## Summary

This release adds a manual "try to keep following" action on top of fresh clear
follow route-memory. If a player is following a target and recently witnessed
that target leave through an ordinary visible exit, the player can explicitly
try to take that remembered direction.

This is still not auto-follow. The player is never moved merely because the
target moved.

## Technical Notes

- The follow-step action reads the latest follow route-memory `WorldEvent` for
  the player and accepts only fresh `visible_move` markers with `visibility=clear`.
- The memory must match the current `PlayerFollowIntent`, the player's current
  location and an ordinary visible exit.
- Accepted movement is submitted through the existing movement handler path; it
  does not update player location directly.
- `FOLLOW_ROUTE_STEP_MEMORY_TTL_MS` controls the freshness window and defaults
  to ten real-time minutes.
- No Prisma migration was added.

## Out Of Scope

- No continuous follow loop.

- No group/party travel.

## Risks / Watchpoints

- Follow-step relies on recent `WorldEvent` route-memory markers. If those
  markers are pruned aggressively, the manual action may become unavailable
  sooner.
- The action sends a short intent line before the existing movement queue
  feedback, so Telegram output is deliberately two-step for now.
