# 0.15.33 - Herald news update notices

Date: 12026-06-04

## Summary

This release adds an admin-facing Herald check for deployed `news.md` entries that do not yet have an active queued or published `HeraldPublication`.

## Changes

- Added `/news_updates` and `/check_news_updates` for the Boundary Mark Chancery Herald bot.
- The check reuses the shared `news.md` parser and archive ordering, then compares content hashes against the durable publication outbox.
- Missing or canceled entries are shown with compact previews and ready `/news_archive_preview N` / `/news_archive_post N` commands.
- Already queued or published entries are counted but not proposed as new work.
- Public `/chronicles` now renders by the internal Chornolis world clock instead of grouping entries by real `Europe/Kyiv` dates.
- Added scribe/admin `/chronicles_real` for the old exact real-time view and `/chronicles_backfill_players` for idempotently reconstructing missing arrival chronicle rows from `Player.createdAt`.
- Added Herald chronicle relay support with `HERALD_CHRONICLE_RELAY_*` env configuration, restart-safe relay markers and admin `/chronicles_pending`, `/publish_chronicles` and `/chronicles_backfill_players` commands.
- Herald archive ordering now prefers semantic release versions before source dates, keeping version-like history in numeric release order.
- Re-queue/manual archive preparation can repair stale `archiveOrder` values on existing matching `contentHash` rows.
- Added the admin-only confirmed `/forget_published_news` recovery command for deleting published `NEWS_MD` / `NEWS_MD_ARCHIVE` outbox rows without deleting Telegram channel messages.
- Added smoke coverage for missed-news detection, HTML escaping and admin-only command listing.
- Player movement notices now respect the observer's own active torch light, so a lit observer should see visible arriving/leaving players by name instead of as anonymous `Хтось`.
- Visible hunter/creature attack text now uses better prey cases and can name a visible hunter in lit scenes instead of falling back to anonymous `Щось`.
- The Telegram menu `Репліки` button is now shown only to Scribes/admins, and direct `/chat`, slashless `chat` aliases and stale chat pagination callbacks use the same Scribe/admin access gate.
- Private speech delivery for greetings, whispers and direct replies now checks session presence at send time, so AFK or ended-session recipients are not nudged by immediate Telegram messages.
- Living target action keyboards now include `Слідувати`, reusing the existing `/follow <target>` follow-intent path for visible players, creatures and local characters.
- `/follow` with no target now reports the current follow intent, marks last-seen targets clearly and shows how to change or clear the trace.
- `/follow_step` dark-memory refusal copy now explains that lighting a torch later helps fresh `/track` searches or new observation, not the old non-directional memory.

## Risks

- This is a manual operator check, not an automatic startup or scheduled notification. The Herald will not publish anything unless an admin explicitly queues or posts it.
- If a news entry is edited after a publication, its content hash changes and the Herald may correctly treat the edited entry as needing review.
- `/forget_published_news confirm` is intentionally sharp: it deletes published news/archive outbox rows so they can be queued again. It does not remove Telegram messages, so any replay creates new Telegram posts and timestamps.
- If an archive backlog was already queued with old `availableAt` spacing, run `/backfill_news_reschedule_pending 13m` after deploy to reapply the repaired release order to pending entries.
- Player movement notices are now per-recipient for player movers; non-player movement batching still uses the existing location-wide light snapshot.
- Creature attack notices still use a shared location message rather than fully per-recipient rendering; this patch names visible hunters and fixes cases without redesigning combat visibility.
- Existing Telegram clients may briefly retain an old reply keyboard with `Репліки`, but pressing it now hits the same Scribe/admin access guard.
- Dark follow-route memories intentionally remain non-directional even if the player finds light later; this keeps `/follow_step` from becoming retroactive auto-follow, but players must use `/track` or fresh observation to recover direction.
- Deferred private-message recap is still future `SES-003` work: this patch prevents immediate pushes while the recipient is away, but does not yet store those lines for delivery on return.
- Public chronicle world-time labels are reconstructed from current `WorldState` and event `createdAt` because old `WorldEvent` rows do not store a persisted absolute world minute. This is display-only and can shift if the world clock is manually reset.
- Herald chronicle relay uses `WorldEvent` markers for idempotency. If old world events are aggressively pruned, relay dedupe markers could disappear and need a more durable table later.
- Historical player backfill is explicit, not automatic; run it only when the owner wants older arrivals added to the chronicle stream.
