# 0.15.34 - Post-0.15.33 regression polish

Date: 12026-06-04

## Summary

This release is a narrow stabilization pass after the broad 0.15.33 slice. It adds regression coverage around follow intent, visibility labels, Herald news checks and chronicle relay state without adding new gameplay behavior.

## Changes

- Added a source-level regression check that the living-target `Слідувати` keyboard callback routes through the shared follow-intent helper instead of a separate path.
- Added direct `actorLabelFromVisibility(...)` coverage for lit observers, fully visible scenes and dark/unlit fallback labels.
- Added Herald news-update checks for all important publication states: missing, pending, published and canceled.
- Added chronicle relay disabled-state coverage so a disabled or misconfigured relay keeps a clear diagnostic reason.
- Kept the existing 0.15.33 runtime behavior unchanged unless those focused tests expose future regressions.

## Risks

- This release mostly strengthens tests. It does not add a broader runtime rewrite for follow, visibility, private speech, Herald publishing or chronicle relay.
- The `Слідувати` callback regression is a source guard; if the callback is intentionally refactored later, the test should move with the new shared follow-intent entrypoint.
- Herald news-update coverage checks publication status classification, not a live Telegram post.
- Chronicle relay remains env-gated and marker-based; a future durable relay table may still be useful if relay history becomes more important.
