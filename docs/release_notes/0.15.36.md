# 0.15.36 - Attention-gate carried-light polish

Date: 12026-06-04

## Summary

This release stabilizes the first MAP-004 attention-gated root gap after the 0.15.35 proof slice. It verifies and hardens the intended carried-light behavior without adding another gated place or a broader visibility framework.

## Changes

- Made the root-gap guard explicitly accept a player-carried active lit torch when location visibility alone would still be dark.
- Kept ordinary local light support through the existing location visibility rules.
- Clarified the darkness refusal copy for `/crawl`: light a torch or find another light source, then inspect the gap again.
- Added a short success lead-in so direct `/crawl` still feels like the character notices the gap in the light.
- Expanded `scripts/test/attention-gated-location.cjs` to cover dark/no-light, carried-lit-torch and local-light reveal behavior.

## Risks

- This is a narrow helper for the first authored root gap, not a general skill-gated exit framework.
- Carried-light proof is helper-level rather than a full database integration playthrough with real torch rows.
- Future viewer-specific light work may still want a broader visibility helper, but this patch deliberately avoids that wider refactor.
