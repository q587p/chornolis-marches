# 0.15.37 - Track-aware attention-gated passage

Date: 12026-06-05

## Summary

This release adds the second MAP-004 runtime proof: a small waking-world passage that is revealed by fresh tracks or clear follow route-memory, not by another light-only gate.

## Changes

- Added a tiny hidden grass-run micro-location near the early dry meadow.
- Added an inspectable `Прим’ята трава` feature that stays vague without a fresh trace and reveals `Пройти за слідом` (`/follow_trace`) only when the player has fresh visible tracks or a recent clear follow-route memory in that location.
- Added a hidden `INSIDE` entry and visible `OUTSIDE` return without exposing the entry as an ordinary exit list item.
- Added `/follow_trace` plus English and Ukrainian text aliases for the one-off track-gated passage action.
- Added focused coverage for no-loot/no-resource boundaries, hidden-exit leak prevention, track/follow-memory reveal decisions and aliases.

## Risks

- This is still an authored MAP-004 proof, not a generic gated-location framework.
- Fresh-track detection uses existing `WorldTrack` rows and current track visibility; if track retention or visibility rules change, the gate cadence may need tuning.
- Hidden water-word route memory is intentionally excluded from this gate, so unusual hidden passages still require their own learning/trigger rules.
