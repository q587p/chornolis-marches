# 0.15.38 — Freshening learning and skill tuning

This release makes freshening the next bounded skill-effect slice after gathering, without pulling attack or full combat forward.

## Added

- Freshening observation now writes canonical `CharacterLearningProgress` rows with:
  - `skillKey = freshening`;
  - `sourceKey = observation`;
  - `contextKey = freshening`.
- Player freshening can receive a small bounded success-chance bonus from stored freshening progress.
- Practice and observation both contribute to the first freshening effect through the canonical learning table.
- Added focused tests for:
  - canonical freshening observation progress input;
  - level-zero unchanged chance;
  - higher-level chance improvement;
  - capped chance behavior;
  - meat yield remaining unchanged.
- Added planning docs for the next learning sequence and future content tasks:
  - cooking observation/effect parity;
  - observed actor skill level helper;
  - attack canonical learning bridge without combat effects;
  - post-combat training arena;
  - horses and mounted travel;
  - regional active curiosities;
  - post-dream questlets;
  - daily/weekly world tasks.

## Changed

- `freshenCorpseForMeat(...)` accepts an optional success chance bonus while keeping the default species-based chance unchanged.
- The player action completion path computes the freshening bonus only for player freshening.
- Meat yield, corpse decay, hunter/NPC freshening and public skill UI remain unchanged.

## Risks

- Freshening observation dedupe still depends on existing WorldEvent source markers and retention.
- The first effect aggregates all canonical `freshening` rows for the same context; if later food learning splits contexts more finely, this helper may need a more detailed policy.
- The bonus is conservative and capped; live play may show that the exact tuning should move up or down.
