# 0.15.39 — Cooking learning and skill tuning

This release gives cooking the same small bounded learning/effect shape as the previous freshening slice, without adding recipes, food quality or economy effects.

## Added

- Cooking observation now writes canonical `CharacterLearningProgress` rows with:
  - `skillKey = cooking`;
  - `sourceKey = observation`;
  - `contextKey = cooking`.
- Player cooking can receive a small bounded success-chance bonus from stored cooking progress.
- Practice and observation both contribute to the first cooking effect through the canonical learning table.
- Added focused tests for:
  - canonical cooking observation progress input;
  - level-zero unchanged chance;
  - higher-level chance improvement;
  - capped chance behavior;
  - cooking failure remaining possible.

## Changed

- `meatCookingSucceeds(...)` accepts an optional success chance bonus while keeping the default base chance unchanged.
- The player cooking path computes the cooking bonus only from stored player learning progress.
- Raw meat cost, cooked meat yield, hunger relief, recipes, stations and public skill UI remain unchanged.

## Risks

- Cooking observation dedupe still depends on existing WorldEvent source markers and retention.
- The first effect aggregates all canonical `cooking` rows for the same context; if later food learning splits contexts more finely, this helper may need a more detailed policy.
- The bonus is conservative and capped; live play may show that the exact tuning should move up or down.
