# 0.15.40 — Actor learning foundation

This release makes canonical learning actor-aware without changing combat, attack effects or broad NPC behavior.

## Added

- Added persistent `CreatureLearningProgress` storage alongside existing player `CharacterLearningProgress`.
- Added actor-aware learning helpers for:
  - player and creature actor refs;
  - recording actor learning progress;
  - reading actor learning rows;
  - qualitative learning summaries;
  - raw technical learning rows;
  - conservative `observedActorSkillLevel(...)` defaults.
- Bridged supported creature gathering practice into canonical creature learning for herbs, berries and mushrooms.
- Added qualitative learning display for ordinary `/me` and target inspection surfaces.
- Added raw learning rows to technical details and extended `/learning` so scribes can inspect `creature #id` rows.
- Added focused tests for actor helper routing, qualitative labels, technical output and creature gathering practice.

## Changed

- Existing player learning helpers and player learning rows remain compatible.
- Player gathering, freshening and cooking effects are unchanged.
- Creature gathering still uses the same action path; this release only lets safe supported practice leave a canonical learning trace.

## Risks

- Creature learning starts with a separate table to avoid risky player-table migrations; future broad actor learning may revisit that storage shape.
- Profession/species defaults in `observedActorSkillLevel(...)` are conservative estimates, not a full NPC skill tree.
- The first creature bridge covers only supported non-combat gathering practice; observation, attack and broader NPC learning remain future slices.
