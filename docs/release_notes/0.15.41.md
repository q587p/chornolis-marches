# 0.15.41 - NPC observation learning MVP

This release adds the first narrow NPC/creature observation-learning path on top
of the `CreatureLearningProgress` storage from `0.15.40`.

## Added

- Added `src/services/creatureObservationLearning.ts` for tightly scoped
  creature observation learning.
- Creature `LOOK` completion can now scan recent local gathering source events
  and record canonical creature `gathering` / `observation` progress for
  supported herbs, berries and mushrooms.
- Added `Creature gathering observation` `WorldEvent` markers to dedupe one
  creature observing one source event.
- Added focused regression coverage for:
  - observer eligibility;
  - gathering source parsing;
  - unsupported source filtering;
  - same-source dedupe;
  - multiple eligible creatures observing one source;
  - canonical creature learning rows.

## Changed

- Creature observation is deliberately profession-gated. Herbalists and hunters
  can learn from this narrow supported gathering observation path; ordinary
  animals do not gain broad gathering/cooking/freshening learning.
- Technical learning surfaces from `0.15.40` can show creature observation rows
  once they exist.

## Not Changed

- No attack canonical bridge yet.
- No attack success or damage effects.
- No broad NPC observation scheduler.
- No public skill sheet or raw progress UI.
- No freshening/cooking creature observation bridge in this slice.

## Risks

- Observation dedupe depends on `WorldEvent` markers. If old events are pruned
  aggressively, old source observations may become repeatable sooner than
  intended.
- The trigger uses existing creature `LOOK` completion, so live cadence depends
  on current NPC look behavior.
- The first observation bridge covers only supported gathering sources; broader
  food or combat observation should remain focused follow-ups.
