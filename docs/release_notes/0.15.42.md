# 0.15.42 - Attack canonical learning bridge

This release moves the existing attack practice/observation bridge into
canonical actor learning storage without changing combat mechanics.

## Added

- Added canonical player and creature `attack` / `practice` / `attack`
  learning progress for eligible completed attack attempts.
- Added canonical player `attack` / `observation` / `attack` learning progress
  for existing attack observation through `LOOK` / `INSPECT` surfaces.
- Added a conservative hunter-only creature attack observation bridge through
  existing creature `LOOK` completion.
- Added `Attack canonical practice` and `Attack canonical observation`
  `WorldEvent` markers for source tracking and dedupe.
- Added focused tests for:
  - player attack practice rows;
  - creature attack practice rows;
  - player attack observation rows;
  - hunter-only creature observation eligibility;
  - same-source observation dedupe;
  - no success/damage modifier leakage from learning helpers.

## Preserved

- Existing attack hit/miss/damage behavior is unchanged.
- Weapon behavior is unchanged.
- Predator and hunter attack AI is unchanged.
- PvP and broader combat remain out of scope.
- The older attack kill observation/milestone flavor remains in place.

## Risks

- Observation dedupe depends on `WorldEvent` source markers and retention.
- Canonical practice currently treats misses, wounds and kills as one learning
  amount each. Future tuning may split outcomes if live cadence needs it.
- Creature attack observation is intentionally conservative. Broader predator or
  animal learning should be a separate focused pass.
