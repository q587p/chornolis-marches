# 0.15.43 - Tracking practice and track-reading quality

Date: 12026-06-05

## Summary

`0.15.43` turns executed `/track` actions into canonical tracking practice and adds the first small output-quality effects for stored tracking progress.

This is not auto-follow, route replay, hidden passage repetition or a broad tracking skill tree.

## Added

- Canonical `tracking` / `practice` / `track_search` progress for executed player `/track` actions.
- Canonical `tracking` / `practice` / `track_detail` progress for detailed track inspection.
- Canonical `tracking` / `practice` / `track_gate` progress when a player successfully uses the track-gated grass-run passage.
- A small `Tracking practice` `WorldEvent` marker for local source context.
- Focused helper coverage for tracking practice input, storage, bounded quality effects, rough age labels and darkness-safe output.

## Changed

- Tracking level 2+ can show rough qualitative age labels in brief `/track` output.
- Tracking level 3+ can show a few more matching tracks, capped at 12.
- Tracking level 4+ can sense that track signs exist in darkness, but still cannot read direction or identity without light.
- Follow-intent track prioritization remains unchanged.

## Preserved

- `/track` still spends stamina and respects existing visibility rules.
- Darkness still hides direction and identity.
- Hidden water-word route memory does not become track direction.
- No automatic movement, continuous following, route replay or hidden-route repeat was added.
- No public `/skills` UI or combat effect was added.

## Risks

- Executed `/track` records practice even when no track is found, because searching terrain noise is part of learning; live tuning can revisit the amount.
- Rough age labels may make brief track output busier for experienced trackers.
- Dedupe/cooldown was not added to `/track` practice in this slice; repeated paid searches can train tracking slowly.
