# 0.15.44 - Learning surfaces and planning cleanup

Date: 12026-06-06

## Summary

`0.15.44` stabilizes learning display after the recent freshening, cooking, actor, attack and tracking learning run.

It also closes the clearly completed LEARN planning items whose acceptance was satisfied by merged `0.15.21` and `0.15.38`-`0.15.42` work.

This is a UX/consistency and planning-cleanup pass. It does not add new skills, public `/skills`, attack modifiers, map content or new skill effects.

## Changed

- Ordinary `/me` learning summaries stay capped and qualitative.
- Visible target inspection now shows only a small number of meaningful stored learning rows, instead of mixing in profile defaults as public `Навички:` text.
- Technical target inspection separates stored learning rows from profile defaults.
- Admin `/learning` output uses the same technical row formatter for players and creatures, including the no-rows case.
- The learning-effect ordering policy is now documented: the current action should use previously stored experience, then record practice for future attempts unless a slice explicitly says otherwise.

## Preserved

- Raw learning keys, ids, totals and source/context details remain technical-only.
- `/track` keeps using previous experience for the current read and records practice afterward.
- Profile defaults remain available to technical/debug surfaces as conservative estimates.
- No public skill sheet was added.
- No combat, attack chance, damage or weapon behavior changed.
- No broad umbrella/testing cleanup was done outside the listed learning items.

## Risks

- Some local-character inspections may show fewer `Навички:` lines until actual stored learning reaches the visible threshold.
- Profile defaults are still conservative regex-style estimates and should move to a small data/config map if they spread.
- Live play should still watch whether target inspection feels too much like a stat sheet.
- Planning cleanup status depends on the current MVP acceptance, not on future broader skill-system ambitions.
