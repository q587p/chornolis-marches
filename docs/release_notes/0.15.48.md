# 0.15.48 - Learning lookup and WorldEvent performance indexes

Date: 12026-06-06

## Summary

- Fixed the scribe-only learning lookup so named creature/local-character targets can be resolved, not only `creature #id`.
- Separated stored creature learning rows from profile/default estimates in the admin learning output.
- Added pragmatic `WorldEvent` lookup indexes for recent learning, follow, chronicle and cooldown/dedupe query patterns.
- Added lightweight slow-path timing logs for common latency hotspots without logging private message text.

## Boundaries

- No gameplay features, group movement, combat effects, public skill sheet, map content or learning mechanics were added.
- `WorldEvent.description` marker searches remain a known future performance risk; this release only adds safe B-tree indexes around title/location/player/time pruning.
