# 0.15.8 - Polish, quieter movement and survival edges - 12026-06-02

This patch collects small player-facing polish, movement-noise reduction and survival edge fixes for the current `0.15.x` ecology/survival pass.

## Summary

- Added a temporary raw mushroom poison bridge: eating carried `mushrooms` now has a 1-in-10 risk of a bad bite.
- A bad mushroom consumes the item, lowers HP by roughly one third without dropping the character below 1 HP in this first slice, and increases hunger by `+1`.
- Documented the future mushroom-variety and poison/effect system so this blunt bridge can later become inspectable, learnable and less random.
- Tightened corpse visibility so darkness hides corpse target lists, old corpse action buttons, text-target lookup, bulk pickup and queued freshening completion.
- Stopped completed freshening source messages from being rewritten to the old generic “prey freshened” summary after the newer weapon-aware result message is sent.
- Coalesced visible non-player/non-animal creature movement notifications per location behind a short runtime window.
- Added `NON_PLAYER_MOVEMENT_NOTIFICATION_WINDOW_MS`, defaulting to 60 seconds, for tuning the movement coalescing window.
- Coalesced movement messages now use one compact text block with one tracks button and deliberately omit arrival target buttons in the first slice to avoid stale target buttons.
- Added first animal-directed speech reactions for targeted `SAY` / `WHISPER`: mice/rabbits can flee through ordinary movement, foxes show wary attention, and wolves can warn with a growl.
- Added text-command parity for handmade campfire actions: `/build_campfire`, `/light_campfire`, `/douse_campfire`, `/dismantle_campfire`, with Ukrainian and MUD-style aliases.

## Player-facing

- Mushrooms usually still ease hunger a little.
- Rarely, a mushroom can taste wrong, hurt the character and make hunger sharper.
- Corpses need light like other ground objects: if a torch or fire goes out, the player must bring light back before inspecting, picking up or freshening them.
- Fast non-player/non-animal movement in one location should arrive as one compact local message instead of a stack of repeated arrival/departure messages.
- Speaking or whispering directly to animals can now produce small species-shaped reactions; this is not taming or full conversation.
- Handmade campfire actions are no longer button-only: players can type `скласти вогнище`, `підпалити вогнище`, `погасити вогнище`, `розібрати вогнище`, or the matching slash/MUD-style forms.

## Risks / Rollback

- The mushroom risk is intentionally simple and random until the future identification/poison system replaces it.
- This first slice avoids dropping HP below 1 from mushroom poisoning alone; if full poisoning should knock characters out later, that should go through the ordinary condition/effect pipeline.
- Completed freshening now only removes stale source buttons instead of rewriting the old source message text; if Telegram refuses to edit markup on an old message, stale buttons may remain best-effort as before.
- The movement coalescer is an in-memory debounce. If the process restarts during the window, pending flavor lines may be dropped; world state and tracks are unaffected.
- The first coalescing slice omits target buttons from movement messages. Players can still use `/look` / `/examine` to see who is present.
- Animal speech reactions are intentionally small and not taming/conversation. Wolf escalation and richer context remain in `ANIMAL-001` follow-up work.
- No-target campfire text commands choose the first suitable nearby handmade campfire for that state. If several handmade campfires pile up in one location, direct feature-inspection buttons remain the more precise control.
- Rollback: revert the mushroom-use helper, freshening source-message cleanup, notification-buffer/action-completion changes, animal speech helper and docs/news entries if the slice feels worse in live play.
