# 0.15.8 - Polish and survival edges - 12026-06-02

This patch collects small player-facing polish and survival edge fixes for the current `0.15.x` ecology/survival pass.

## Summary

- Added a temporary raw mushroom poison bridge: eating carried `mushrooms` now has a 1-in-10 risk of a bad bite.
- A bad mushroom consumes the item, lowers HP by roughly one third without dropping the character below 1 HP in this first slice, and increases hunger by `+1`.
- Documented the future mushroom-variety and poison/effect system so this blunt bridge can later become inspectable, learnable and less random.
- Tightened corpse visibility so darkness hides corpse target lists, old corpse action buttons, text-target lookup, bulk pickup and queued freshening completion.

## Player-facing

- Mushrooms usually still ease hunger a little.
- Rarely, a mushroom can taste wrong, hurt the character and make hunger sharper.
- Corpses need light like other ground objects: if a torch or fire goes out, the player must bring light back before inspecting, picking up or freshening them.

## Risks / Rollback

- The mushroom risk is intentionally simple and random until the future identification/poison system replaces it.
- This first slice avoids dropping HP below 1 from mushroom poisoning alone; if full poisoning should knock characters out later, that should go through the ordinary condition/effect pipeline.
- Rollback: revert the mushroom-use helper and docs/news entries if the risk feels too punishing before the identification layer exists.
