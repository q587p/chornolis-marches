# 0.16.17 — Sleep comfort and field command polish

Date: 12026-06-09
Branch: `codex/0.16.17-sleep-comfort-predator-polish`

## Summary

- Added direct `Лягти` (`/lie`) and `Сон` (`/sleep`) actions to the starter cellar `Сухі лежанки` inspection.
- Added a bounded feature-level sleep comfort signal so the cellar bunks and active campfires make ordinary sleep stamina recovery modestly faster than bare sleep.
- Changed the ordinary sleep auto-wake hint to use a Ukrainian hour phrase, not a numeric clock label.
- Allowed foxes, hawks, owls and snakes to use authored `UP` exits while keeping ordinary animal vertical movement guarded.
- Fixed generic predator observer text so lower-case animal labels are capitalized at sentence starts and predator feeding messages use per-observer visibility instead of a hardcoded `Щось`.
- Added Cyrillic `афк` as a session-presence alias for `/afk`.
- Registered restored command surfaces for visible play: `/signals`, `/socials`, `/track_fox`, `/attack_all` and `/kill_all`; `/freshen` remains a direct/news-clickable command without being added to the Telegram side command menu.
- Added a bulk attack queue-cap preflight so `/attack_all` / `/kill_all` either queues the full matching target set or fails before enqueueing anything.
- Added scribe/admin `/deleteCampfire` handling for removing ordinary non-magical campfires from the current or named location.
- Let supported profession NPCs dismantle strange totems through the shared dismantle/twig recovery behavior, without adding a new reward path.
- Adjusted strange-totem spawn boundaries to use protected regions/special areas instead of over-broad per-location exclusions, so ordinary eligible riverbank locations can participate.
- Updated the carried-torch fading warning and follow-up action so players are nudged to light another torch while fire remains, when they carry one.
- Documented that population-floor restoration remains species-level, not a per-location guarantee.

## Technical Notes

- `sleep_comfort_multiplier` stays bounded and feature-local. It does not add a new sleep mode, shelter system or broad comfort taxonomy.
- Active campfires still keep their existing temporary stamina-cap behavior and now also share the modest ordinary-sleep comfort path.
- Species-specific vertical movement depends on authored vertical exits and the allowlist of fox/hawk/owl/snake. It does not add group movement or a new animal pathfinder.
- Creature move completion reloads the exit destination before rechecking animal route boundaries, so planned species-specific vertical moves and completed queued moves use the same guard.
- Predator feeding notifications now use `notifyLocationDynamic(...)` plus the same observer light/visibility label path as visible player movement, so a player with a lit torch can see the predator label while an unlit observer can still receive the atmospheric fallback.
- `афк` now routes through the same shared alias/session-presence handling as `afk`, so it sets AFK directly instead of falling through to the unknown-input helper.
- Bulk attack keeps the existing attack action semantics and target visibility rules. The new preflight compares the full target count with `MAX_QUEUED_ACTIONS_PER_ACTOR` before the first enqueue because the first queued attack intentionally interrupts the previous player queue.
- Creature totem dismantling reuses the same dismantle completion path and twig resource grant shape as player dismantling, but without player reward text.
- The strange-totem spawn change keeps starter camp, dream tutorial, closed gate and old-bridge protected regions excluded; it narrows the protection boundary so `under_bridge_18_05` can behave like riverbank where appropriate.

## Risks / Rollback

- Existing production worlds need seed/map refresh after deploy for the starter cellar bunk metadata to appear.
- This is not a broad sleep-system rewrite: no new danger wakeups, shelter map, lucid dream behavior, NPC sleep loop or public sleep-quality numbers.
- This is not a broad animal movement rewrite: foxes, hawks, owls and snakes get only species-specific vertical movement, not group movement or a new AI pathfinder.
- `/attack_all` and `/kill_all` are queue helpers for existing attackable visible targets, not a combat-system rewrite.
- NPC totem dismantling is a profession action hook only; it is not a broad profession/economy system.
- No Prisma schema, migrations, WorldEventMarker, fauna seed group, Lisovyk restoration, `/spirit`, mentorship, group movement, broad combat rewrite, economy or profession-system changes.
- Rollback is a normal revert of the release commit; if already deployed, refresh the live seed/map afterward so authored feature metadata mirrors the reverted code.
