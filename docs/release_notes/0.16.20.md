# 0.16.20 — Action queue observability and safe nudge

Date: 12026-06-09
Branch: `codex/0.16.20-action-queue-observability`

## Summary

- Added runtime snapshots for the action queue loop: running state, last start/finish/error, last pass counts and phase timings.
- Measured `processActionQueue(...)` phases: player completion, player queued-action start, keyboard refresh, creature queue kick and total pass time.
- Extended `getActionQueueStats()` with player/creature queue splits, overdue splits, oldest queued ages and the top five overdue running actions.
- Added guarded scribe/admin `/queueDebug` and slashless `queueDebug` / `queuedebug`.
- Added guarded scribe/admin `/queueNudge` and slashless `queueNudge` / `queuenudge`; it requests one pass through the existing overlap guard and does not create a new interval.

## Technical Notes

- `processActionQueue(...)` still does not call full `recoverStamina(bot)` work; recovery remains in the separate recovery loop from 0.16.18.
- `/queueDebug` is read-only diagnostics.
- `/queueNudge` is not a cleanup command. It does not cancel, pause, clear or reorder queue rows.
- Protected `/health` JSON receives the expanded queue stats through the existing status data path.

## Validation

- `node scripts/test/action-queue-performance.cjs`
- `npm.cmd test`
- `npm.cmd run typecheck`
- `npm.cmd run build`
- `git diff --check`

## Risks / Rollback

- The richer stats add a few extra diagnostic reads to status/debug paths, not to gameplay command execution.
- The nudge command depends on the in-process bot instance; before the action queue loop has started it reports that no pass was requested.
- No Prisma schema, migrations, WorldEventMarker, action duration, stamina cost, sleep comfort, combat, tracking result, movement, predator, social alias, totem behavior or other 0.16.17/0.16.18 gameplay-surface changes.
- Rollback is a normal revert of the release commit.
