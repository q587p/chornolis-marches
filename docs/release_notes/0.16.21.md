# 0.16.21 — Creature queue backpressure

Date: 12026-06-09
Branch: `codex/0.16.21-creature-queue-backpressure`

## Summary

- Added a conservative creature queue backpressure planner.
- Added creature queue runtime snapshots for selected mode/reason, completion/start timings, completed/started creature counts and skipped-start state.
- Integrated player-first creature queue backpressure: overdue player actions keep player completion/start first, while creature completion is limited and queued creature starts are paused by default.
- Extended `/queueDebug` with creature queue mode, reason, timings and counts.
- Renamed the admin debug keyboard/hears label to `🧵 Службова черга` / `Службова черга` to avoid colliding with ordinary player queue wording.

## Technical Notes

- Defaults are intentionally conservative and tunable by environment:
  - `CREATURE_QUEUE_BACKPRESSURE_ENABLED=true`;
  - `CREATURE_QUEUE_BACKPRESSURE_PLAYER_OVERDUE_MS=1000`;
  - `CREATURE_QUEUE_BACKPRESSURE_PAUSE_STARTS_MS=5000`;
  - `CREATURE_QUEUE_BACKPRESSURE_RUNNING_BATCH=50`;
  - `CREATURE_QUEUE_BACKPRESSURE_COMPLETION_CONCURRENCY=5`;
  - `CREATURE_QUEUE_BACKPRESSURE_START_BATCH=0`.
- `processActionQueue(...)` still processes player completion/start first and still does not call full `recoverStamina(bot)`.
- Creature backpressure does not cancel, delete, reorder or reprioritize queued creature actions. It only limits due creature completion work and can skip starting queued creature actions while players are overdue.
- `/queueNudge` remains a guarded request for one pass through the existing overlap guard; it does not create a new interval.

## Risks / Rollback

- If live queue pressure shows creature work is too conservative, tune the backpressure environment values rather than changing gameplay semantics.
- Queued creature actions can wait longer when players are overdue; they are not lost or mutated.
- No Prisma schema, migrations, action duration, stamina cost, player queue semantics, tracking result, movement, combat, predator, sleep, social alias, totem or WorldEventMarker behavior changes.
- Rollback is a normal revert of the release commit.
