# 0.16.23 — Action completion latency observability

Date: 12026-06-10
Branch: `codex/0.16.23-action-completion-latency-observability`

## Summary

- Added sanitized runtime observations for individual action completion duration, overdue age and outcome.
- Instrumented player and creature completion loops without changing action semantics or queue ordering.
- Extended guarded `/queueDebug` with recent slow completions and completion errors.
- Added focused tests and 0.16.23 release surfaces.

## Technical Notes

- Completion observations record only action id, actor type, player/creature id, action type, duration, overdue age, outcome and compact error text.
- Observations do not include raw action payloads, Telegram chat ids, player text, whispers or secrets.
- `ACTION_COMPLETION_SLOW_MS` can override the slow-completion threshold; otherwise it follows `SLOW_COMMAND_LOG_MS` and defaults to `1000`.
- `ACTION_COMPLETION_SAMPLE_LIMIT` controls the recent slow/error ring buffers, defaulting to `10` and capped at `50`.
- `slow:actionCompletion` logs only slow completions and errors.

## Risks

- This is observability only: no gameplay behavior, action duration, stamina/HP, queue ordering, dedupe, recovery, backpressure, movement, combat, tracking, sleep, social, predator, totem, Herald archive, Prisma schema or migration changes.
- Telegram sends remain awaited as before; this release adds visibility first.
- If slow completions point to notification-heavy branches, use that evidence for a later send/fan-out observability or throttling PR rather than changing completion semantics here.
