# 0.16.24 - Telegram send latency observability

Date: 12026-06-10
Branch: `codex/0.16.24-telegram-send-observability`

## Summary

- Added sanitized runtime observations for Telegram `sendMessage` duration, outcome and compact error context.
- Routed central notification fan-out paths through observed `safeSendMessage` labels.
- Extended guarded `/queueDebug` with Telegram send totals, slow send samples and send error samples.
- Added admin-only Herald archive tail continuation commands that queue only safe missing tail gaps detected by `/archive_republish_gap`.
- Added focused tests and 0.16.24 release surfaces.

## Technical Notes

- Telegram send observations record only developer-chosen context labels, duration, outcome, compact sanitized error text and safe reply-markup presence.
- Observations do not include Telegram chat ids, usernames, message text, payloads, tokens, secrets, raw Telegram responses, parse-mode content, player speech or whispers.
- `TELEGRAM_SEND_SLOW_MS` can override the slow-send threshold; otherwise it follows `SLOW_COMMAND_LOG_MS` and defaults to `1000`.
- `TELEGRAM_SEND_SAMPLE_LIMIT` controls recent slow/error samples, defaulting to `10` and capped at `50`.
- `safeSendMessage` still returns Telegram messages on success, returns `null` for blocked-user errors and rethrows other errors.
- Notification fan-out remains serial and awaited.
- `/archive_republish_continue_preview <interval>` and `/archive_republish_continue <interval>` reuse the deployed `news.md` parser, oldest-first archive order and gap detection. They refuse internal holes, queue only missing tail entries, use `archive-continue:*` content hashes, preserve `archiveOrder` and normal archive rendering, and do not resume the publisher loop.

## Risks

- No gameplay behavior, queue ordering, action duration, stamina/HP, recovery cadence, creature backpressure, Telegram delivery semantics, Prisma schema or migration changes.
- Slow-send samples identify hot fan-out paths but do not throttle, batch, parallelize or suppress sends.
- Herald archive tail continuation is admin-only, refuses internal archive holes and does not publish immediately or unpause the publication queue.
