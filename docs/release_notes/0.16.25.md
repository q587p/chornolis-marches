# 0.16.25 - Direct Telegram send observability and admin/status polish

Date: 12026-06-10
Branch: `codex/0.16.25-direct-telegram-send-observability`

## Summary

- Added a semantics-preserving `observedSendMessage(...)` helper for direct Telegram sends.
- Routed selected action completion, action recovery and torch-fading direct sends through sanitized Telegram send telemetry.
- Kept `safeSendMessage(...)` blocked-user behavior unchanged and preserved direct-send throw semantics.
- Added admin-only `/stat_species` plus the `🐾 Види` admin-menu button for a compact animal species block.
- Changed hunger/fatigue guidance copy from `У Речах є...` to `Серед речей є...`.
- Added planning follow-ups for low-light gather/search warnings and social-signal posture/catalog gates.
- Added focused tests and 0.16.25 release surfaces for the expanded scope.

## Technical Notes

- `observedSendMessage(...)` calls `bot.api.sendMessage(...)` directly and returns the Telegram message on success.
- If Telegram reports a blocked-user 403, `observedSendMessage(...)` records outcome `blocked` and rethrows the original error object.
- If Telegram throws any other error, `observedSendMessage(...)` records outcome `error` and rethrows the original error object.
- `safeSendMessage(...)` keeps its existing behavior: blocked-user errors return `null`, while other errors rethrow.
- Observations use stable low-cardinality context labels such as `actionCompletion.track.reply`, `actionRecovery.sleepAutoWake` and `fire.fadingTorchNotice`.
- Observations do not include Telegram chat ids, usernames, message text, payloads, tokens, secrets, raw Telegram responses, parse-mode content, player speech or whispers.
- `/stat_species` reuses the existing ecology stats data and lists all animal species rows by `CreatureSpecies.kind = ANIMAL`, including species with zero current animals.
- `GATHER-001` and `SOC-011` are planning-only follow-ups; they do not implement low-light gathering penalties or social posture gates in this release.
