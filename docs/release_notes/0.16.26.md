# 0.16.26 - Deferred non-critical Telegram follow-ups

Date: 12026-06-10
Branch: `codex/0.16.26-deferred-telegram-followups`

## Summary

- Added a bounded in-process deferred Telegram follow-up queue for explicitly non-critical messages.
- Started the deferred follow-up loop from the game bot startup path only; Herald startup remains unchanged.
- Added guarded `/queueDebug` diagnostics for deferred follow-up pending/sent/dropped/expired/error counts and sanitized recent context rows.
- Migrated only optional tutorial/guidance/recovery-extra sends after the primary response path is already handled.
- Added focused deferred Telegram tests and 0.16.26 release surfaces.

## Deferred Follow-Ups

The first conservative migration covers:

- tutorial action hints after movement/look/examine primary replies;
- first-night guidance after the location view has already been sent;
- mentorship practice prompts and herbalist brewing hints after the lesson text;
- idle reminder voice comments from the recovery loop;
- daypart hints after ordinary sleep auto-wake;
- tutorial full-rest flavor comments after the actual recovery notice.

## Still Awaited

These paths remain direct/awaited:

- primary action completion replies;
- action failure replies;
- movement destination and location-view primary messages;
- fatigue, hunger, knockout and sleep auto-wake notices;
- queue-control replies;
- admin and Herald replies;
- normal notification fan-out paths already using `safeSendMessage(...)`.

## Technical Notes

- `deferTelegramFollowup(...)` is best-effort and bounded by environment knobs:
  - `DEFERRED_TELEGRAM_ENABLED`, default `true`;
  - `DEFERRED_TELEGRAM_POLL_MS`, default `250`;
  - `DEFERRED_TELEGRAM_MAX_PENDING`, default `200`;
  - `DEFERRED_TELEGRAM_MAX_PER_PASS`, default `5`;
  - `DEFERRED_TELEGRAM_MAX_AGE_MS`, default `300000`.
- Deferred delivery uses `observedSendMessage(...)`, so existing Telegram send telemetry still records delivery duration and outcome.
- If the queue is disabled, full, expired or delivery fails, the deferred path records sanitized counters and does not throw into action completion or recovery.
- Diagnostics do not expose Telegram chat ids, usernames, message text, payloads, parse-mode content, player speech, whispers, tokens, secrets or raw Telegram API responses.

## Risks And Rollback

- Deferred follow-ups are intentionally best-effort. A full or disabled queue can drop optional follow-ups, but primary action and recovery replies remain direct.
- The queue is global serial in this PR; there is no per-chat parallel delivery.
- Reverting this PR restores the previous awaited follow-up sends.
- No action durations, stamina/HP math, queue ordering, action dedupe, creature backpressure, recovery cadence, movement, combat, tracking, sleep, social, predator, totem, Herald behavior, Prisma schema or migrations changed.
