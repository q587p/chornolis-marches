# 0.16.27 - Prisma query latency observability

Date: 12026-06-10
Branch: `codex/0.16.27-prisma-query-observability`

## Summary

- Added sanitized runtime observations for Prisma model operation duration, outcome and compact error context.
- Extended guarded `/queueDebug` diagnostics with `databaseQuery`, `recentSlowDatabaseQueries` and `recentDatabaseQueryErrors`.
- Added `slow:databaseQuery` warnings for slow or failed Prisma model operations.
- Added focused database-query diagnostics tests and 0.16.27 release surfaces.

## Technical Notes

- The shared Prisma client now uses a central query extension for model operations.
- `DATABASE_QUERY_OBSERVABILITY_ENABLED` controls whether observations are recorded; default is `true`.
- `DATABASE_QUERY_SLOW_MS` controls slow-query sampling/logging; it falls back to `SLOW_COMMAND_LOG_MS`, then `100`.
- `DATABASE_QUERY_SAMPLE_LIMIT` controls how many recent slow/error samples guarded `/queueDebug` keeps, defaulting to `10` and capped at `50`.
- Slow and error rows include model, action, duration, outcome and compact sanitized error text only.
- Diagnostics do not expose query arguments, SQL, parameters, Telegram chat ids, message text, payloads, private whispers, tokens, secrets or database URLs.

## Risks And Rollback

- This is observability only: no query shape, index, queue ordering, action duration, stamina/HP math, recovery cadence, backpressure, Telegram/deferred behavior, Herald behavior, Prisma schema, migrations or gameplay rule changes.
- Raw query calls outside Prisma model operations are not broadened in this PR.
- Reverting this PR removes the Prisma model operation diagnostics and restores the previous shared Prisma client construction.
