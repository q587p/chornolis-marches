## Summary

- Added an operator triage guide for collecting a 10-15 minute live validation window after deploy.
- Added a paste-friendly performance observation template for idle, active and after-drain guarded queue snapshots.
- Added a pure `summarizeQueueDebugText(...)` helper for pasted debug text and a focused regression test.
- Added a planning watchpoint requiring evidence before queue, recovery, Telegram, database or backpressure tuning PRs.
- Added 0.16.28 release surfaces.

## Analysis Workflow

The new guide asks maintainers to collect:

- an idle baseline snapshot;
- an active repeated-action snapshot, especially around quick tracking clicks;
- an after-drain snapshot;
- matching slow-log excerpts from the same time window.

The decision tree maps existing telemetry to one narrow next PR category:

- database/index/query tuning;
- recovery pruning;
- Telegram delivery or deferred-follow-up tuning;
- creature backpressure threshold tuning;
- action-specific optimization;
- another observation window first.

## Helper

`summarizeQueueDebugText(...)` is intentionally pure. It accepts a pasted text
block and returns headline counts plus recommendations. It does not call the
database, does not require Telegram, does not store the supplied text and does
not run in the live command path.

## Risks And Rollback

- This is documentation/read-only analysis support. It changes no gameplay,
  queue ordering, action durations, stamina/HP math, recovery cadence, creature
  backpressure thresholds, Telegram send semantics, deferred follow-up policy,
  Herald archive behavior, Prisma schema, migrations, SQL indexes or query
  behavior.
- Rollback is a normal revert of the release commit.
