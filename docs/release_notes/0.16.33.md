# 0.16.33 - Animal movement observation tracking hint

Date: 12026-06-11
Branch: `codex/0.16.33-animal-movement-observation`
Status: proposed

## Summary

- Added one bounded animal-movement observation cue for tracking: visible rabbit movement under good light can record a small qualitative tracking observation for nearby players.
- Reused existing animal movement, track creation, visibility rules and learning/progression storage instead of adding a new observation command or schema.
- Kept the feedback rare through one-species scope plus per-player duplicate/cooldown checks.
- Fixed generic predator-kill observer wording so prey species names stay lowercase inside the second sentence: `миша` and `жаба`, not `Миша` or `Жаба`.
- Fixed animal age labels so grammatical adjective gender follows the displayed species noun: `доросла жаба` and `доросла змія`, while sexed labels such as `дорослий миш` still work.
- Fixed animal social-signal flee wording so species names are capitalized at sentence start: `Жаба лякається жесту й кидається геть.`
- Moved `TRACK-LEARN-001-B` to `testing`, regenerated planning exports, and cleaned `docs/planning/next.md` so completed `WPN-003` work no longer leads the immediate-candidate list.

## Risks And Rollback

- Main risk is making the cue feel too tutorial-like or too farmable; tests cover rabbit-only eligibility, visibility guardrails, duplicate/cooldown bounds and no raw skill numbers.
- This is one observation pattern only. It does not implement the full following skill, change follow-assist behavior, bypass hidden routes or darkness, add auto-navigation, group movement, combat changes, scheduler/runtime tuning, action queue/recovery/Telegram/Prisma tuning, schema changes or migrations.
- Rollback is a normal revert of this narrow observation-learning PR.
