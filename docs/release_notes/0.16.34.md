# 0.16.34 - Learning visibility guardrails

Date: 12026-06-11
Branch: `codex/0.16.34-learning-visibility-guardrails`
Status: proposed

## Summary

- Added a shared visibility helper for learning from visible observations.
- Blocked `/track` practice learning when tracks are present but unreadable in darkness.
- Kept rabbit movement observation aligned with the same visibility rules while preserving the existing nearby-detail plus track-readability requirement.
- Guarded existing local look/inspect observation learning for attack, gathering, freshening and cooking so poor visibility does not grant progress for unseen cues.
- Registered safe main-bot deep links for public news/web/Herald references to `/sleep_tutorial`, `/follow_assist_on`, `/make_tincture` and `/brew_tincture`.
- Made technical location details show base danger separately from temporary local tension when recent attacks or crowding raise effective danger.
- Stopped follow-assist catch-up from queuing another movement once the follower has reached the target's current location, without deleting the route memory or footprints.
- Split character-card HP recovery estimates into passive, active-rest and near-campfire lines so they match the stamina recovery breakdown.
- Added a world tick runtime watchdog snapshot with running state, current/last phase, phase durations, stale warning text and skipped already-running attempts.
- Made manual `/tick` report an already-running world tick instead of pretending a new pass ran successfully.
- Moved `TRACK-LEARN-001-C` to `testing`, updated `docs/planning/next.md`, regenerated planning exports and bumped package metadata to `0.16.34`.

## Risks And Rollback

- Main risk is accidentally suppressing a learning cue that should still work in dim but readable conditions or adding noisy scribe/admin diagnostics; focused tests cover dark-blocked, light-enabled, reached-target, recovery text and world tick runtime-reporting cases.
- This is guardrails, diagnostics and polish only. It does not implement the full following skill, bypass hidden routes, add auto-navigation, group movement, combat mechanics, ecology math changes, creature AI changes, the `0.17` scheduler rewrite, action queue/recovery cadence changes, Telegram delivery rewrites, Prisma tuning, public skill/progress UI, schema changes or migrations.
- Rollback is a normal revert of this narrow guardrail PR.
