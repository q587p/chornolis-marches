# 0.16.7 - Grammar agreement helper cleanup and archive republish queue

## Summary

- Added shared grammar agreement helpers for player/creature gender and gendered word selection.
- Replaced duplicated local gender helpers in `targets.ts` and `playerText.ts`.
- Expanded Ukrainian grammar tests for player/creature gender fallback, `wordByGender` and representative visible strings.
- Added admin-only Herald archive republish controls that queue deployed `news.md` archive entries oldest-first through the durable `HeraldPublication` outbox.
- Archive republish rows use a distinct `NEWS_MD_ARCHIVE_REPUBLISH` source type and `republish:v1:*` content-hash namespace, so existing published archive rows do not block the rebuild run.
- Republished archive posts render like normal archive posts (`📜 З архіву Канцелярії`) and do not use `/repost_publication` formatting or the explicit repeated-publication disclaimer.

## Admin commands

- `/archive_republish_preview [interval]`
- `/archive_republish_queue [interval]`
- `/archive_republish_status`
- `/archive_republish_cancel`

The republish interval defaults to 30 minutes when omitted. Invalid interval text returns a visible admin reply instead of queueing anything.

## Risks

- This is a helper cleanup only; player-facing text is intended to stay unchanged.
- The Herald republish queue is admin/operator-facing. It does not add gameplay behavior and does not change `/repost_publication`.
- `/archive_republish_cancel` cancels only unpublished `NEWS_MD_ARCHIVE_REPUBLISH` rows; normal pending news/archive rows and already published history are left intact.
- WorldEvent/WorldEventMarker logic, Prisma schema and migrations are unchanged.
- Broader text surfaces such as social signals, keyboards, help text, location prose and action completions remain future focused slices.
