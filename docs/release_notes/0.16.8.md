# 0.16.8 - Social signal content extraction and scribe suggestions

## Summary

- Added `src/content/social/socialSignals.ts` for social signal labels and message templates.
- Kept `src/services/socialSignals.ts` focused on target resolution, notification sending, animal/NPC reactions, world/audit event creation and creature `currentAction` updates.
- Preserved the current social signal labels and player-facing messages.
- Added focused regression coverage for social IDs, labels, representative player-target, creature-target and targetless messages, and quick-social target defaults.
- Added scribe/admin-only unknown-command suggestions for service commands, so `/addl` can suggest the lit-torch helper for scribes without exposing admin commands to ordinary players.
- Added alias regressions so suggested exact inputs such as `kill` and documented personal-summary inputs such as `skills` route to existing responses instead of falling back to unknown-command text.
- Shortened public `/help` by removing one-off or highly specific command blurbs, and clarified that AFK starts automatically after 13 minutes of inactivity.
- Expanded strange-totem dismantle feedback to a stable set of 13 atmospheric aftermath lines.
- Updated follow-assist hints to use clickable `/follow_assist_on` and `/follow_assist_off` commands while keeping the older `/follow_assist on/off` forms compatible.
- Changed legacy `auto` / `/auto` input to show Spirit Call status and controls instead of immediately enabling the helper; the later explicit control surface uses `/spirit_on` and `/spirit_off`, while bare `/spirit` shows status.
- Added admin-only Herald publication queue diagnostics so scribes can distinguish all unpublished PUBLIC rows, ready-now rows, future scheduled rows and per-source outbox state.
- Clarified `/pending_publications` as ready-now output and added `/future_publications` for the next scheduled rows.

## Risks

- This is intended as content/suggestion/help cleanup only; WorldEvent/WorldEventMarker logic, Prisma schema, migrations, Herald archive republish, gameplay rules, keyboards, location prose and action completions are unchanged.
- HTML escaping remains in the service layer; content templates receive already escaped actor and target forms.
- Admin command suggestions are gated through `isScribeAdmin(...)`; command routing and admin access enforcement are unchanged.
- Herald queue diagnostics are admin-only and do not publish, cancel or reschedule outbox rows.
- Future social signal expansion or broader text cleanup should remain separate.
