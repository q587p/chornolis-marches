# 0.16.9 — Visible NPC learner loop MVP

Date: 12026-06-08

## Summary

- Added one narrow living-world proof that local NPCs can also learn through attention.
- Added Radana, a local learner near the starter border camp.
- Radana uses a neutral `local_person` species with a `profession_learner` marker, not the herbalist or hunter specialist species.
- The learner can quietly stay near, follow or watch a matching specialist profile such as an herbalist or hunter.
- Location and inspection text can subtly show that the learner is watching a specialist work or keeping to a specialist's trail.
- No room notifications are emitted for the learner loop.
- Telegram side-command descriptions now use distinct icons for `/help` and `/respawn`.
- The ordinary `☰ Меню` keyboard is the same for players and scribes; scribe-only statistics stay in `🛠 Адмін меню` and direct command surfaces.

## Technical Notes

- Added `src/services/npcLearner.ts` with table-driven teacher profiles for herbalist and hunter specialists.
- Moved learner profile copy and stable learner action phrases into `src/content/npc/npcLearnerProfiles.ts`; the learner service keeps behavior, routing, queueing and persistence.
- Moved `/help` and `/commands` authored pages into `src/content/help/helpText.ts`; the Telegram handler keeps callbacks, keyboards and send logic.
- Learner progress is bounded technical marker state in `currentAction`, hidden from public location text.
- The learner stops seeking the teacher once the profile-supported level is reached, then rests and decays enough for the loop to repeat later.
- The loop does not add Prisma schema, migrations, WorldEventMarker changes, player rewards, public skill UI, economy, combat, mentorship, group movement or `/spirit` teacher unlocks.
- Лісовик learning remains future work and is unchanged.
- Added workflow guardrails: ordinary PRs target `main`, release surfaces must name their version, and full `npm.cmd test` runs should use a longer timeout as the test manifest grows.

## Validation

- Added `scripts/test/npc-learner-loop.cjs`.
- Extended creature-observation and herbalist helper coverage where the learner loop touches existing observation/specialist surfaces.
- Extended the test manifest so the learner loop runs in `npm test`.
- Added or extended focused coverage for help content boundaries, Telegram command-menu icons and ordinary-menu consistency.

## Risks / Watchpoints

- The first learner now has a neutral human-like species archetype. Keep future learner behavior keyed to the `profession_learner` marker/profile, not to specialist species keys.
- The visible surface is deliberately subtle; broader teaching, apprenticeship and profession behavior should remain separate future slices.
- Future fisher or other specialist teachers should be added as new learner profiles, not as a broad economy or profession rewrite.
