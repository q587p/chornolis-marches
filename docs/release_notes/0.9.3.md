# 0.9.3 — AdminHelp, persistent auto and location feature visibility

## Summary

This update repairs the admin/debug command surface, makes auto-mode persistent between process updates, and makes interactive location features visible immediately in the location view.

## Gameplay / UX

- `📍 Локація` now includes feature buttons immediately.
- `🧍 Персонаж` now includes auto-mode status and an inline auto toggle.
- `☰ Меню` contains news/help/back only; auto-mode is no longer presented as a general menu item.
- `/auto` and `/autoStop` still work as text commands.
- Manual `/tick` now replies with the latest world tick counters.

## Technical

- Added `Player.isAutoEnabled`.
- Added persistent auto timer restore on startup.
- Added reset-time auto-state cleanup.
- Added a small manual tick report handler registered before the existing tick command registration.
- Kept the old full admin/debug command list while adding `/reset` to it.

## Planning

- Admin/debug permissions are explicitly tracked as future work.
- Advanced auto-mode profiles and profession-aware automation remain future work.
